# 월간 해커톤 : 투자 데이터를 시각화하라- Skills 기반 대시보드 설계

분석 가이드(MD)를 설계하고 바이브 코딩으로 투자 분석 대시보드를 구현하는 해커톤

## 대회 개요
금융 기업에서는 다양한 투자 데이터를 분석하여 시장 상황을 파악하고, 이를 직관적으로 확인할 수 있는 대시보드를 빠르게 구축해야 합니다. 그러나 데이터 구조가 매번 다르고 분석 기준도 달라, 일관된 방식으로 투자 데이터를 시각화하고 해석하기 어려운 문제가 있습니다.

이번 해커톤에서는 투자 분석 규칙을 정의한 Skills.md 문서를 설계하고, 이를 기반으로 바이브 코딩을 활용하여 금융 투자 대시보드 웹 서비스를 구현합니다.

참가자는 투자 데이터 분석 기준, 시각화 방식, 인사이트 표시 규칙을 정의하고, 이를 바탕으로 다양한 투자 데이터를 효과적으로 보여줄 수 있는 **데이터 기반 투자 대시보드 서비스**를 개발해야 합니다.

_※ 본 대회에서 말하는 Skills는 특정 서비스의 고유 기능만을 의미하지 않으며, 분석 규칙과 생성 지침을 담은 범용 MD 문서를 포괄적으로 의미합니다._

Skills.md 문서는 다음 역할을 합니다.

(예시)

- 투자 데이터 분석 기준 정의
- 투자 지표 계산 규칙 정의
- 시각화 선택 기준 정의
- 리포트 구성 흐름 설계
- 투자 인사이트 생성 규칙 정의

이를 통해 참가자는 일관된 분석 기준을 기반으로 다양한 투자 데이터를 대시보드 형태로 시각화할 수 있는 범용 금융 분석 시스템을 설계하게 됩니다.

## 평가 기준
### 1차 평가 : 대중 투표

- 최종 제출한 참가팀을 대상으로 대중 투표를 진행하여 상위 10팀 선정
    - 제출팀 가중치 : 60%
    - 참가팀 가중치 : 20%
    - 대중투표 가중치 : 20%

※ 동점자 처리 기준
- 참가팀 투표 점수가 높은 팀 우선
- 산출물 최초 업로드 시간이 빠른 팀 우선
### 2차 평가 : 내부 심사위원 평가

- 대중 투표 상위 10팀을 대상으로 내부 심사위원 정성 평가 진행
- 최종 순위는 내부 심사위원 평가 점수로 결정
- 평가기준

| 평가항목         | 평가 내용                              | 배점  |
| ------------ | ---------------------------------- | --- |
| 범용성          | 투자 데이터 구조 대응 가능성, 재사용 가능성 등        | 25  |
| Skills.md 설계 | 분석 규칙 명확성, 시각화 기준, 인사이트 생성 규칙 등    | 25  |
| 대시보드 자동 생성   | 자동 분석 동작 여부, 시각화 적절성, UI 구성 완성도 등  | 25  |
| 바이브코딩 활용     | 문서 기반 생성 구조, 자동 생성 수준, 수동 구현 최소화 등 | 15  |
| 실용성 및 창의성    | 실제 활용 가능성, 확장 기능, 사용자 경험 개선 등      | 10  |
| 합계           | -                                  | 100 |
## 규칙/가이드라인

### 제출물 규칙

1. **기획서 : PDF**
- 참가자는 구현할 투자 대시보드 서비스 기획서를 제출합니다.
- 포함 내용 : 서비스 개요, 분석 흐름 설계, 대시보드 구성, Skills.md 설계 방향, 확장 기능 아이디어 등

2. **Skills.md 문서 : .md (PDF로 변환하여 제출)**

- 투자 분석 규칙을 정의한 문서를 제출합니다.
- ※ Skills.md는 1개 또는 여러 개 제출할 수 있습니다.  
    ※ 여러 개 제출할 경우, 기능별·역할별로 구분하여 구성할 수 있습니다. (.zip파일로 제출)
    - 예시 포함 내용 : 데이터 분석 규칙 정의, 시각화 선택 기준, 대시보드 구성 규칙 등

3. **웹 페이지 : 웹 링크**
- Skills.md를 기반으로 바이브 코딩을 활용하여 투자 대시보드 웹 서비스를 구현합니다.
    - 배포 URL (필수)
    - Github 저장소 링크 (선택)

## 개발/배포 규칙
- 배포 URL은 외부에서 접속 가능해야 하며 심사 기간 동안 접근 가능해야 합니다.
- 기술 선택은 자유입니다.
- 참가자는 더미 데이터 또는 공개 데이터를 활용하여 구현할 수 있습니다.
    - 예시 데이터) 주가 데이터, ETF 데이터, 포트폴리오 데이터, 재무 지표, 시장 지표, 거래 데이터 등
- 외부 API 사용 시 심사자가 별도 키 없이 확인 가능해야 합니다.

## 공정성/저작권

- 제출물(코드/이미지/폰트/아이콘 등)은 저작권·라이선스를 준수해야 합니다.
- 타인의 결과물을 무단 복제/도용하거나 라이선스를 위반한 경우 실격 또는 감점될 수 있습니다.
- 팀 내 역할/분업은 자유이며 생성형 AI 도구 사용은 허용합니다.

## 심사 관련 유의사항

- 심사는 기본적으로 배포 URL을 기준으로 진행합니다.
- 확인이 어려운 기능은 심사에서 제외될 수 있습니다.
- 운영진의 추가 안내가 있을 경우 공지 내용이 우선 적용됩니다.
