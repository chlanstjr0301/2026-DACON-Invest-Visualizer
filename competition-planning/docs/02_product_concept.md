## 1. 제품명

**Skillfolio**

---

## 2. 한 줄 정의

Skillfolio는 투자 전문가와 포트폴리오 분석 실무자가 업로드한 비정형 투자 CSV/XLSX 파일을, Skills.md에 정의된 금융 분석 규칙에 따라 자동으로 처리하여 재현 가능한 포트폴리오 분석 대시보드와 애널리스트 형식의 리포트로 변환하는 서비스다.

---

## 3. 핵심 가치 제안

각 가치 제안은 docs/01_problem_definition.md에서 정의한 Pain Point와 직접 연결된다.

| # | 가치 제안 | 연결된 Pain Point | 설명 |
|---|---------|-----------------|------|
| V1 | 비정형 CSV를 그대로 업로드해도 분석이 시작된다 | P1: 파일을 올려도 분석이 바로 시작되지 않는다 | AI 컬럼 매핑이 사용자의 CSV 컬럼명을 date, ticker, return, weight 등 분석에 필요한 의미 단위로 자동 추론한다. 사용자는 데이터 변환 작업 없이 분석을 시작할 수 있다 |
| V2 | 데이터, 컬럼 매핑, Skills.md 규칙이 동일할 때 수치 지표와 대시보드 구성이 재현된다 | P2: 같은 데이터를 넣어도 분석 결과가 달라진다 | 분석은 Skills.md에 명시된 규칙(MDD 계산 기준, 변동성 정의, 집중도 임계값 등)에 따라 수행된다. 동일한 입력 데이터, 사용자가 확정한 컬럼 매핑, 동일한 Skills.md 규칙이 주어지면 수치 지표와 대시보드 구성 규칙이 재현된다. 재현 범위는 Skills.md로 정의된 결정적(deterministic) 계산 구간에 한정된다 |
| V3 | 데이터가 부족하면 어떤 분석이 가능하고 무엇이 필요한지 명확히 안내한다 | P3: 데이터가 부족할 때 시스템이 오류를 내거나 침묵한다 | 데이터 충분성 검사 단계에서 요청된 분석별로 가용 여부를 판단하고, 불가능한 분석에 대해서는 필요한 추가 데이터를 사용자에게 안내한다. 부분 분석이 가능한 경우 가능한 범위에서 대시보드를 생성한다 |
| V4 | 분석 근거가 대시보드와 함께 표시된다 | P4: 차트를 만들었지만 왜 그런 결과가 나왔는지 설명하기 어렵다 | 각 분석 결과는 어떤 Skills.md 규칙에 따라 산출되었는지를 Skills 로그로 기록하고 대시보드에 함께 표시한다. 분석 근거를 추적할 수 있어 보고서 첨부에 신뢰성이 생긴다 |
| V5 | 대시보드와 애널리스트 형식의 리포트가 동시에 생성된다 | P5: 분석 결과를 바로 보고서로 쓸 수 없다 | 대시보드 생성과 동시에 각 지표의 해석과 위험 요인을 포함한 애널리스트 형식의 리포트가 화면 내 패널에 표시된다. MVP에서는 Markdown/텍스트 형태의 리포트를 제공하며, PDF/PNG 내보내기는 P2 확장 기능으로 분류한다 |

## 4. 서비스 작동 흐름

사용자 관점의 단계별 흐름이다. 기술 구현 방식이 아니라 사용자가 경험하는 순서를 기술한다.

### 정상 흐름 (데이터가 충분한 경우)

```
[1단계] 파일 업로드
사용자가 CSV 또는 XLSX 파일을 업로드한다.
시스템은 파일을 수신하고 컬럼 목록과 데이터 미리보기를 표시한다.

[2단계] AI 컬럼 매핑 확인
시스템이 컬럼명을 자동으로 추론하여 분석 의미 단위(date, ticker, weight, return, price, quantity 등)에 매핑한다.
사용자는 매핑 결과를 확인하고 필요시 수정한다.

[3단계] 분석 목적 명확화
사용자가 분석 목적을 선택하거나 자연어로 입력한다.
예: "포트폴리오 성과와 리스크를 분석해줘", "비중 집중도를 점검해줘"

[4단계] 데이터 충분성 검사
시스템이 요청된 분석별로 필요한 컬럼과 데이터 기간이 충족되는지 판단한다.
분석 가능 목록과 가능하지 않은 분석 목록을 사용자에게 표시한다.

[5단계] 대시보드 자동 생성
Skills.md에 정의된 규칙에 따라 분석을 수행하고 대시보드를 자동 생성한다.
화면은 포트폴리오 요약, 성과 차트, 드로다운 차트, 비중 분포, 리스크 지표 테이블로 구성된다.

[6단계] 애널리스트 리포트 확인
대시보드 옆 패널에 각 지표에 대한 애널리스트 형식의 해석과 위험 요인이 표시된다.
Skills 로그를 통해 각 분석이 어떤 규칙에 따라 수행되었는지 확인할 수 있다.

[7단계] 리포트 확인 및 저장
대시보드 및 리포트를 화면에서 확인한다. Markdown 또는 텍스트 형식으로 저장할 수 있다.
PDF/PNG 내보내기는 P2 확장 기능으로, MVP에서는 제공하지 않는다.
```

### 예외 흐름 (데이터가 불충분한 경우)

```
[4단계] 데이터 충분성 검사 → 일부 분석 불가 판단

시스템 출력 예시:
  - 가능한 분석: 성과 분석, 드로다운 분석, 집중도 분석
  - 불가능한 분석: 벤치마크 비교 (벤치마크 수익률 데이터 없음)
  - 제한적 가능 분석: 리스크 분석 (샤프 지수 계산에 무위험 수익률 데이터 필요)

사용자 선택지:
  1. 가능한 분석만으로 대시보드 생성 (권장)
  2. 추가 데이터 파일을 업로드하고 재검사
  3. 분석 목적을 변경

[5단계 이후] 가능한 분석 범위로 대시보드 생성
불가능한 분석 항목은 대시보드에 "데이터 부족 — [필요한 데이터명]이 필요합니다" 형태로 명시된다.
```

## 5. 핵심 차별화

> **면책 표기:** 이하 비교는 각 경쟁 서비스의 공개된 제품 설명을 기반으로 한 기획 해석이다. Skillfolio의 설계 선택을 명확히 하기 위한 대조적 서술이며, 경쟁 서비스의 공식 결함 선언이 아니다.

| 비교 대상 | 공개 정보 기반 특징 | Skillfolio의 설계 대조 |
|---------|-----------------|---------------------|
| Portfolio Visualizer, Koyfin, Sharesight (금융 특화 서비스) | 분석 품질과 지표 신뢰성이 높음. 각 서비스의 공개 기능 설명에 따르면 데이터 진입 경로는 티커 입력, 브로커 연동, 또는 자체 포트폴리오 등록 방식이 주를 이룸 | Skillfolio는 사용자가 보유한 비정형 CSV를 그대로 업로드해 분석하는 것을 주된 진입 경로로 설계한다. 금융 분석 품질을 유지하면서 입력 유연성을 확보하는 것이 설계 목표다 |
| Power BI Copilot (범용 BI + AI) | CSV 업로드와 자연어 대시보드 생성을 지원함. Microsoft 공식 문서는 Copilot 생성 보고서가 비결정적이며 도메인 전문가를 대체할 수 없다고 명시함 | Skillfolio는 금융 포트폴리오 분석에 특화된 규칙을 Skills.md에 사전 정의하고, 그 규칙이 분석을 제어하게 한다. 자연어 프롬프트가 아니라 규칙 명세가 결과를 결정하므로 재현 가능한 출력을 설계 목표로 한다 |
| Julius AI (범용 AI 데이터 분석) | CSV 업로드와 자연어 분석을 지원하는 범용 도구로 자체 설명함. 공개된 제품 기능에서 Skills.md 형태의 규칙 명세 구조가 전면에 제시되는 방식은 확인되지 않음 (기획 해석) | Skillfolio는 포트폴리오 분석 규칙을 Skills.md에 사전 정의하고 그 규칙이 분석 전체를 제어하도록 설계한다. 사용자가 매 세션마다 분석 기준을 재정의하지 않아도 일관된 분석이 수행되는 것이 설계 목표다 |
| 모든 경쟁 서비스 공통 | 분석 규칙이 사용자에게 명시적으로 공개된 문서 형태로 제공되는 구조가 확인되지 않음 | Skillfolio는 Skills.md를 서비스의 분석 명세서로 공개하고, 대시보드 결과가 어느 규칙에서 비롯되었는지 Skills 로그로 추적 가능하게 한다 |

## 6. MVP 범위

### 6.1 지원 입력 형식

| 형식 | 지원 여부 |
|------|---------|
| CSV | 지원 |
| XLSX | 지원 |
| 실시간 API 연동 | 미지원 (MVP 외) |
| 브로커 연동 | 미지원 (MVP 외) |

### 6.2 지원 데이터 유형

| 데이터 유형 | 예시 컬럼 구조 |
|-----------|-------------|
| 포트폴리오 보유 종목 (holdings) | 종목명 또는 티커, 보유 수량 또는 비중, 평가금액 |
| 일별 수익률 (daily returns) | 날짜, 종목명 또는 포트폴리오 식별자, 수익률 |
| 일별 자산 가격 (daily asset prices) | 날짜, 종목명, 종가 |
| 단순 거래 내역 (simple transaction history) | 날짜, 종목명, 매매구분, 수량, 가격 |

### 6.3 핵심 분석 항목

| 분석 항목 | 주요 지표 | 필요 데이터 |
|---------|---------|-----------|
| 성과 분석 | 누적 수익률, 기간 수익률, CAGR | 일별 수익률 또는 일별 가격 |
| 드로다운 분석 | 최대낙폭(MDD), 드로다운 차트, 회복 기간 | 일별 수익률 또는 누적 수익률 시계열 |
| 리스크 분석 | 변동성(연환산), 샤프 지수, 최대낙폭 | 일별 수익률 |
| 비중 분석 | 종목별/자산군별 비중 분포 | 보유 종목 비중 또는 평가금액 |
| 집중도 분석 | 상위 N개 종목 비중 합산, Herfindahl 지수 | 보유 종목 비중 |
| 벤치마크 비교 | 초과수익, 추적오차, 정보비율 | 사용자 파일에 벤치마크 데이터가 포함된 경우에 한정 |

### 6.4 출력물

| 출력물 | 형태 | 설명 |
|-------|------|------|
| 대시보드 | 웹 화면 | 성과 차트, 드로다운 차트, 비중 분포, 리스크 테이블로 구성된 자동 생성 레이아웃 |
| 애널리스트 리포트 | 텍스트 패널 / Markdown | 각 지표의 해석, 위험 요인, 주요 발견 사항을 포함한 구조화된 리포트. PDF/PNG 내보내기는 P2 확장 기능이다 |
| Skills 로그 | 텍스트 | 각 분석 결과가 어떤 Skills.md 규칙에 따라 산출되었는지 기록한 감사 로그 |

## 7. 비목표

다음 항목은 Skillfolio MVP에서 구현하지 않는다. 기획서와 기능 설계에서 언급하거나 암시하지 않는다.

| 비목표 항목 | 이유 |
|-----------|------|
| 실시간 시장 데이터 연결 | 라이선스 비용, 심사 기간 중 API 키 노출 문제, MVP 범위 초과 |
| 자동 거래 / 주문 실행 | 금융 규제, 브로커 연동 복잡도, 공모전 평가 항목과 무관 |
| 세금 계산 및 세금 리포트 | 국가별 세법 적용 복잡도, MVP 집중도 분산 우려 |
| 틱 데이터 / 분봉 분석 | 마이크로스트럭처 분석은 포트폴리오 수준 분석과 목적이 다름 |
| 팩터 회귀 / 스타일 분석 | Fama-French 팩터 분석 등 고급 팩터 리서치는 MVP 범위 초과 |
| 전사 BI 플랫폼 기능 | 범용 데이터 파이프라인, 조직 권한 관리, 대용량 데이터 처리 |
| 브로커 API 연동 | 인증 복잡도, 브로커별 스키마 차이, 심사 기간 중 접근 불가 위험 |
| 단일 종목 분석 | 개별 종목 재무 분석, 밸류에이션, 실적 예측은 포트폴리오 분석과 목적이 다름 |
| 범용 데이터 분석 | "모든 종류의 데이터를 분석한다"는 설계는 이 서비스의 목적이 아님 |
| 자연어만으로 분석 규칙을 결정하는 구조 | 세션 간 재현성을 보장하려면 규칙은 Skills.md에 사전 정의되어야 함 |

## 8. 경쟁 서비스 대비 포지션

두 축을 기준으로 Skillfolio의 포지션을 정리한다.

- **X축 (입력 유연성):** 사용자가 업로드한 비정형 CSV/XLSX를 처리하는 능력
- **Y축 (금융 도메인 특화성):** 포트폴리오 분석에 특화된 규칙과 지표 체계의 내장 수준

```
                    금융 도메인 특화성 높음
                           |
        Portfolio          |          [Skillfolio]
        Visualizer         |       (금융 특화 + 유연 입력)
        Koyfin             |
        Sharesight         |
                           |
---입력 유연성 낮음-----------+----------- 입력 유연성 높음---
                           |
                           |       Power BI Copilot
                           |       Julius AI
                           |       (범용 + 유연 입력)
                    금융 도메인 특화성 낮음
```

**Skillfolio의 포지션 근거:**

| 포지션 축 | Skillfolio의 설계 선택 |
|---------|---------------------|
| 입력 유연성 (높음) | AI 컬럼 매핑이 비정형 CSV/XLSX 컬럼을 분석 의미 단위로 자동 추론한다. 사용자가 데이터 변환 없이 업로드만으로 분석을 시작할 수 있도록 설계한다 |
| 금융 도메인 특화성 (높음) | Skills.md에 포트폴리오 분석 규칙(수익률 계산 기준, MDD 정의, 집중도 임계값, 시각화 선택 기준, 인사이트 생성 규칙)을 사전 정의하고 그 규칙이 분석을 제어한다 |

## 9. 공모전 평가항목과의 연결

| 평가항목 | 배점 | Skillfolio의 대응 설계 |
|---------|-----|----------------------|
| 범용성 | 25 | 포트폴리오 수준의 투자 데이터를 대상으로, 컬럼 구조가 다른 CSV/XLSX를 AI 컬럼 매핑으로 처리하는 설계. Holdings, 일별 수익률, 가격, 거래 내역 네 가지 데이터 유형에 대응하는 분석 흐름을 Skills.md에 정의하여 재사용 가능한 분석 구조를 확보한다 |
| Skills.md 설계 | 25 | 입력 스키마 규칙, 지표 계산 규칙, 데이터 충분성 판단 규칙, 시각화 선택 기준, 인사이트 생성 규칙, 리포트 구성 흐름을 Skills.md에 명시한다. 대시보드 출력 결과가 Skills.md 어느 규칙에서 비롯되었는지 Skills 로그로 추적 가능하게 하여 규칙-결과 연결을 실증한다 |
| 대시보드 자동 생성 | 25 | 파일 업로드 → AI 컬럼 매핑 → 목적 명확화 → 충분성 검사 → 분석 실행 → 대시보드 레이아웃 자동 생성의 전체 흐름을 자동화한다. 사용자가 차트 유형이나 레이아웃을 직접 선택하지 않아도 Skills.md의 시각화 선택 규칙에 따라 적절한 차트가 선택된다 |
| 바이브코딩 활용 | 15 | Skills.md가 대시보드 컴포넌트 생성의 기준 문서로 실제 작동한다. "이 Skills.md를 충족하는 대시보드 컴포넌트를 생성하라"는 구조로 코드 생성을 유도하여 Skills.md → 코드 생성 → 대시보드 출력의 연결을 기획서에서 명시한다 |
| 실용성 및 창의성 | 10 | 데이터 충분성 안내(P3 해소), 분석 근거 추적(P4 해소), 리포트 즉시 생성(P5 해소)은 실무 사용자가 반복적으로 직면하는 마찰을 제거한다. Skills 로그 공개로 분석 투명성을 높이는 것은 금융 분야의 신뢰성 요건에 대응하는 창의적 설계다 |

## 10. 다음 Stage에서 PRD로 확장해야 할 포인트

다음 항목들은 현재 기획 개념 수준에서 정의되었으나, PRD 단계에서 구체적인 수용 기준과 엣지 케이스를 갖춰야 한다.

| 항목 | PRD에서 필요한 구체화 |
|------|-------------------|
| AI 컬럼 매핑 정확도 기준 | 매핑 결과의 신뢰도 임계값, 저신뢰도 컬럼에 대한 사용자 확인 요청 조건, 오매핑 발생 시 사용자 수정 UX |
| 데이터 충분성 판단 규칙의 명세 | 분석 항목별 최소 요건(최소 기간, 필수 컬럼, 최소 종목 수), 부분 분석 가능 조건의 판단 트리 |
| Skills.md 규칙과 대시보드 출력 간 연결 명세 | 각 분석 항목이 어떤 Skills.md 규칙 ID를 참조하는지 매핑 테이블, Skills 로그 포맷 정의 |
| 다중 파일 업로드 처리 방식 | 두 개 이상의 파일을 업로드했을 때 결합 기준(날짜, 종목명), 컬럼 충돌 처리, 파일 간 기간 불일치 처리 |
| 에러 및 데이터 품질 경고 UX | 결측치 비율 임계값, 날짜 형식 오류, 중복 행 감지 시 사용자에게 표시되는 경고 메시지 기준 |
| 비기능 요건 | 파일 크기 상한, 처리 시간 목표(SLA), 동시 사용자 수, 브라우저 호환성, 보안 요건(파일 저장 기간, 개인정보 처리 방침) |
| 사용자 스토리와 수용 기준 | 각 페르소나별 핵심 시나리오에 대한 사용자 스토리 작성, 수용 기준(Acceptance Criteria) 명세 |
| 내보내기 형식 및 리포트 구성 | PDF 내보내기 레이아웃, 리포트 섹션 구성(요약, 원인 분해, 위험 요인, 한계 고지), PNG 해상도 기준 |
