## 섹션 1. 제품 개요

### 1.1 제품명

**Skillfolio**

### 1.2 한 줄 정의

Skillfolio는 투자 전문가와 포트폴리오 분석 실무자가 업로드한 비정형 투자 CSV/XLSX 파일을, Skills.md에 정의된 금융 분석 규칙에 따라 자동으로 처리하여 재현 가능한 포트폴리오 분석 대시보드와 애널리스트 형식의 리포트로 변환하는 서비스다.

### 1.3 핵심 thesis

Skillfolio는 범용 대시보드 도구도 범용 챗봇도 아니다. Skills.md에 정의된 금융 포트폴리오 분석 규칙이 분석 흐름 전체를 제어하고, AI 컬럼 매핑이 비정형 입력 파일의 진입 장벽을 제거하며, 모든 분석 결과가 규칙 추적 가능한 형태로 출력된다.

기존 금융 분석 서비스는 특정 입력 스키마(티커 입력, 브로커 연동, 자체 포트폴리오 등록)를 주된 진입 경로로 설계하는 경향이 있으며, 범용 AI 데이터 분석 도구는 금융 포트폴리오 전용 분석 규칙과 재현성 보장 구조를 전면에 제시하지 않는다. Skillfolio는 이 두 가지 한계를 동시에 다루는 설계 구조를 취하며, 이 구조가 공모전의 범용성·Skills.md 설계·대시보드 자동 생성 평가 항목과 직접 대응한다.

### 1.4 대상 사용자

다음 두 페르소나는 docs/01_problem_definition.md에서 상세 정의되었다.

| 페르소나 | 역할 | 핵심 컨텍스트 |
|---------|------|-------------|
| 페르소나 A: 투자 운용 실무자 | 포트폴리오 분석 담당자, 운용 성과 보고서 작성자, 금융 데이터 실무자 | 월간·분기별 성과 보고서 작성이 필요하며, 데이터는 사내 시스템에서 CSV로 내려받거나 거래 상대방으로부터 Excel로 수수함 |
| 페르소나 B: 개인 퀀트 연구자 / 투자 분석 스터디 참여자 | 퀀트 입문자, 포트폴리오 분석 학습자, 금융 데이터 분석 과제를 수행하는 학생 | 공개 데이터나 직접 수집한 가격·수익률 데이터를 분석하고 그 결과를 시각화해 공유하려 함 |

## 섹션 2. 문제와 배경

### 2.1 해결하려는 문제

금융 실무 현장에서 포트폴리오 데이터는 Excel 또는 CSV 형식으로 수수되는 관행이 지속되고 있다. 이 파일들은 작성 주체에 따라 컬럼명, 날짜 형식, 데이터 구조가 제각각이다. 사용자는 두 가지를 가지고 있다. 데이터와 분석 목적이다. 그러나 현재 사용할 수 있는 도구에서 이 두 가지를 연결하는 과정에는 반복적인 수작업이 개입된다.

### 2.2 사용자 Pain Point 요약

다음 Pain Point는 docs/01_problem_definition.md 섹션 3에서 정의하였다.

| # | Pain Point | 설명 | 해당 페르소나 |
|---|-----------|------|-------------|
| P1 | 파일을 올려도 분석이 바로 시작되지 않는다 | 기존 분석 서비스가 기대하는 스키마와 사용자 CSV의 컬럼명이 다르면 매핑 수작업이 선행되어야 한다 | A, B |
| P2 | 같은 데이터를 넣어도 분석 결과가 달라진다 | 범용 AI 도구는 프롬프트에 따라 결과가 달라질 수 있어 세션 간 재현성이 보장되지 않는다 | A |
| P3 | 데이터가 부족할 때 시스템이 오류를 내거나 침묵한다 | 요청 분석에 필요한 컬럼이 없으면 어떤 분석이 가능하고 무엇이 부족한지 안내받지 못한다 | A, B |
| P4 | 차트를 만들었지만 왜 그런 결과가 나왔는지 설명하기 어렵다 | 분석에 사용된 계산 규칙과 판단 기준을 추적할 수 없어 보고서에 첨부하기 어렵다 | A, B |
| P5 | 분석 결과를 바로 보고서로 쓸 수 없다 | 차트와 수치를 따로 만든 뒤 별도 문서에 붙여넣는 수작업이 필요하다 | A |
| P6 | 데이터 형식마다 분석 흐름을 다시 설계해야 한다 | 일별 수익률 파일과 보유 종목 비중 파일은 구조가 달라 같은 목적이어도 전체 흐름을 새로 구성해야 한다 | A, B |

### 2.3 기존 서비스 gap (기획 해석)

> **면책 표기:** 이하는 각 서비스의 공개된 제품 설명을 기반으로 한 기획 해석이며, 해당 서비스의 공식 결함 선언이 아니다.

| 서비스 유형 | 공개 정보 기반 강점 | 우리의 기획 해석 (빈틈) |
|-----------|------------------|----------------------|
| Portfolio Visualizer, Koyfin, Sharesight 등 금융 특화 서비스 | 높은 분석 품질, 지표 신뢰성. 각 서비스의 공개 기능에 따르면 데이터 진입 경로는 티커 입력, 브로커 연동, 또는 자체 포트폴리오 등록 방식이 주를 이룸 | 사용자 고유의 비정형 CSV/XLSX를 직접 분석 단위로 사용하는 시나리오가 주된 기능으로 문서화되어 있지 않음. 비정형 컬럼 구조의 파일은 별도 변환 없이 사용하기 어려운 구조로 파악됨 |
| Power BI Copilot 등 범용 BI + AI | CSV 업로드와 자연어 대시보드 생성을 지원함. Microsoft 공식 문서는 Copilot 생성 보고서가 비결정적(nondeterministic)이며 도메인 전문가를 대체할 수 없다고 명시 | 금융 포트폴리오 전용 분석 규칙이 내장되지 않으며, 규칙-결과 간 추적 가능한 연결 구조가 전면에 제시되지 않음 |
| Julius AI 등 범용 AI 데이터 분석 도구 | CSV, Excel 업로드와 자연어 분석을 지원하는 범용 도구 | Skills.md 형태의 규칙 명세 구조와 포트폴리오 목적 기반 분석 라우팅, 데이터 충분성 검증 워크플로가 공개된 기능 설명에서 확인되지 않음 |
| 전체 범주 공통 | 각자의 영역에서 완성도 높음 | 분석 규칙이 사용자에게 명시적으로 공개된 문서 형태로 제공되는 구조가 확인되지 않음 |

### 2.4 공모전 주제와의 연결

공모전 공식 명칭은 "월간 해커톤: 투자 데이터를 시각화하라 — Skills 기반 대시보드 설계"이다. 핵심 평가 기준은 Skills.md 규칙 명세 품질과 그 규칙에 근거한 대시보드 자동 생성 동작이다.

Skillfolio가 정의한 문제(비정형 투자 파일을 금융 도메인 규칙으로 처리해 재현 가능한 대시보드로 변환)는 이 평가 구조와 직접 대응한다. P1·P6은 범용성 항목(25점)에, P2·P4는 Skills.md 설계 항목(25점)에, P1·P5·P6은 대시보드 자동 생성 항목(25점)에 각각 연결된다.

## 섹션 3. 제품 목표

### 3.1 사용자 목표

| 목표 | 성공 조건 | 연결 Pain Point |
|------|---------|---------------|
| 비정형 CSV를 업로드하면 분석이 즉시 시작된다 | 컬럼 매핑 수작업 없이 대시보드 생성 단계까지 도달 | P1, P6 |
| 동일 입력에 대해 일관된 분석 결과를 얻는다 | 동일 파일·매핑·Skills.md 규칙에서 수치 지표 재현 | P2 |
| 데이터가 부족할 때 어떤 분석이 가능한지 명확히 안내받는다 | 불가능한 분석에 대해 필요 데이터가 명시된 안내 메시지 수신 | P3 |
| 분석 근거를 추적할 수 있다 | Skills 로그에서 각 지표의 참조 규칙 ID 확인 가능 | P4 |
| 대시보드와 리포트가 동시에 생성된다 | 대시보드 생성과 동시에 화면 내 애널리스트 패널 표시 | P5 |

### 3.2 비즈니스 및 공모전 목표

| 목표 | 측정 기준 |
|------|---------|
| 공모전 2차 심사 진출 | 1차 대중 투표 상위 10팀 내 진입 |
| Skills.md 설계 항목 고득점 | 심사위원이 규칙-결과 연결을 확인할 수 있는 Skills 로그 및 명세 문서 제출 |
| 범용성 항목 고득점 | 컬럼 구조가 다른 4가지 데이터 유형(holdings, 일별 수익률, 일별 가격, 거래 내역) 처리 시연 |
| 대시보드 자동 생성 항목 고득점 | 업로드에서 대시보드 생성까지 사용자 수작업 없이 완료되는 흐름 시연 |

### 3.3 MVP 성공 기준

| 기준 | 측정 방법 | 목표값 |
|------|---------|------|
| 정상 흐름 완주율 | 샘플 CSV 업로드 → 대시보드 생성까지 오류 없이 완주 | P0 데모 데이터셋(5종목 1년치 일별 수익률 CSV) 기준 오류 없이 완주 |
| 데이터 충분성 안내 동작 | 벤치마크 데이터 없는 파일 업로드 시 부족 항목 안내 표시 | 해당 항목 반드시 표시 |
| 컬럼 매핑 고신뢰도 자동 완료 | 표준적인 컬럼명을 가진 파일에서 사용자 수정 없이 매핑 완료 | 신뢰도 0.8 이상 컬럼에 대해 자동 매핑 |
| 대시보드 생성 소요 시간 | 파일 수신부터 대시보드 화면 표시까지 | P0 데모 데이터셋 기준 30초 이내 |
| Skills 로그 표시 | 각 분석 결과에 대해 Skills.md 규칙 ID가 로그에 기록됨 | 모든 P0 결정적 분석 항목에 대해 기록 |
| 분석 결과 재현성 | 동일 입력·매핑·Skills.md 규칙에서 수치 지표 동일 출력 | P0 결정적 분석 항목(MDD, 연환산 변동성, 누적 수익률 등) 기준 동일 결과 출력 |

## 섹션 4. MVP 범위

### 4.1 지원 파일 형식

| 파일 형식 | 지원 여부 | 비고 |
|---------|---------|------|
| CSV (.csv) | 지원 (P0) | UTF-8, EUC-KR 인코딩 처리 포함 |
| XLSX (.xlsx) | 지원 (P0) | 단일 시트 기준. 다중 시트는 사용자가 시트 선택 |
| XLS (.xls) | 미지원 | 구형 형식. 변환 후 업로드 안내 |
| PDF, 이미지 | 미지원 | MVP 외. 문서 파싱은 범위 초과 |
| 실시간 API 연동 | 미지원 | 라이선스·심사 기간 키 노출 문제로 제외 |
| 브로커 연동 | 미지원 | 인증 복잡도 및 MVP 범위 초과 |

### 4.2 지원 데이터 유형

| 데이터 유형 | 예시 컬럼 (사용자 측 비정형 컬럼명 포함) | 비고 |
|-----------|--------------------------------------|------|
| 포트폴리오 보유 종목 (holdings) | 종목명, 티커, 보유수량, 평가금액, 비중(%), 자산군, 업종 | 비중 또는 평가금액 중 하나 이상 필요 |
| 일별 수익률 (daily returns) | 날짜, 종목명, 일별수익률(%), 포트폴리오수익률 | 날짜 컬럼과 수익률 컬럼 필수 |
| 일별 자산 가격 (daily asset prices) | 날짜, 종목명, 종가, 수정종가 | 가격에서 수익률 역산 가능 |
| 단순 거래 내역 (simple transaction history) | 날짜, 종목명, 매매구분(매수/매도), 수량, 가격, 수수료 | MVP에서는 거래 목록 표시 및 보유 내역 요약에 한정. 성과 분석(수익률 계산)에는 일별 수익률 또는 일별 가격 컬럼이 별도로 필요하며, 거래 내역만으로 수익률을 산출하지 않음. 거래 기반 holdings 역산은 P1 기능으로 분류 |

### 4.3 지원 분석 항목

| 분석 항목 | 주요 지표 | 필수 데이터 | 우선순위 |
|---------|---------|-----------|--------|
| 성과 분석 | 누적 수익률, 기간 수익률(1개월·3개월·1년), CAGR | 일별 수익률 또는 일별 가격 시계열 | P0 |
| 드로다운 분석 | 최대낙폭(MDD), 드로다운 차트, 낙폭 회복 기간 | 일별 수익률 또는 누적 수익률 시계열 | P0 |
| 리스크 분석 | 연환산 변동성 (P0), 샤프 지수 (P1, 무위험 수익률 가정 포함) | 일별 수익률 시계열 (최소 30거래일) | P0 / P1 |
| 비중 분석 | 종목별·자산군별·업종별 비중 분포, 파이차트 (트리맵은 P1) | 보유 종목 비중 또는 평가금액 | P0 |
| 집중도 분석 | 상위 N개 종목 비중 합산, Herfindahl-Hirschman Index(HHI) | 보유 종목별 비중 | P1 |
| 벤치마크 비교 | 초과수익(Active Return), 추적오차(Tracking Error), 정보비율(IR) | 사용자 파일 내 벤치마크 수익률 데이터가 존재하는 경우에 한정 | P1 |

> **주의:** 벤치마크 비교는 사용자가 파일 내에 벤치마크 수익률 또는 가격 데이터를 포함시킨 경우에만 수행한다. 외부 시장 데이터 자동 연결은 MVP 범위에 포함하지 않는다.

## 섹션 5. 핵심 사용자 시나리오

### 시나리오 1: 정상 흐름 — 충분한 데이터, 명확한 컬럼 구조

**상황:** 페르소나 A(투자 운용 실무자)가 5개 종목의 1년치 일별 수익률이 담긴 CSV 파일을 가지고 있다. 컬럼명은 Date, Ticker, DailyReturn, Weight처럼 표준에 가깝다.

1. 사용자가 웹 서비스에 접속하여 CSV 파일을 드래그 앤 드롭으로 업로드한다.
2. 시스템이 컬럼 목록(Date, Ticker, DailyReturn, Weight)과 처음 5행의 데이터 미리보기를 표시한다.
3. AI가 컬럼 의미를 추론하여 매핑 결과를 표시한다. Date → 날짜, Ticker → 종목 식별자, DailyReturn → 일별 수익률, Weight → 보유 비중. 모든 컬럼의 신뢰도가 0.8 이상이다.
4. 시스템이 고신뢰도 매핑 결과를 사용자에게 확인 화면으로 표시한다. 사용자는 매핑 내용을 검토하고 이상이 없음을 확인한다.
5. 사용자가 분석 목적으로 "포트폴리오 성과와 리스크 분석"을 선택한다.
6. 시스템이 데이터 충분성을 검사하고 결과를 표시한다. 성과 분석, 드로다운 분석, 리스크 분석(연환산 변동성), 비중 분석이 가능하다. 벤치마크 데이터가 파일에 없으므로 벤치마크 비교는 불가하다고 명시된다. 집중도 분석(HHI)과 샤프 지수는 P1 기능으로 별도 표시된다.
7. 사용자가 "대시보드 생성"을 클릭한다. 30초 이내에 Skills.md 규칙에 따른 대시보드가 자동 생성된다. 누적 수익률 선 차트, MDD 차트, 종목별 비중 파이차트, 연환산 변동성 테이블이 표시된다.
8. 우측 패널에 AI 애널리스트 패널이 표시된다. 각 지표의 해석과 위험 요인이 애널리스트 형식으로 서술되며, 상단에 "이 해석은 AI 보조 생성이며 비결정적입니다"라는 안내가 포함된다.
9. 사용자는 화면 하단의 Skills 로그 패널을 열어 각 지표가 어떤 Skills.md 규칙 ID에 따라 산출되었는지 확인한다.
10. 사용자는 화면 내 리포트 패널에서 분석 결과와 Skills 로그를 확인한다. Markdown 파일 내보내기는 P1 기능으로 이 시나리오에서는 포함하지 않는다.

### 시나리오 2: 데이터 부족 흐름 — 벤치마크 데이터 없음, 수익률 시계열 짧음

**상황:** 페르소나 B(개인 퀀트 연구자)가 3개 종목의 20거래일치 일별 가격 CSV를 업로드하고 성과·리스크 분석을 요청한다.

1. 사용자가 CSV를 업로드한다. 파일에는 Date, Stock, ClosePrice 컬럼이 있다.
2. AI가 컬럼을 추론하고 매핑 결과를 표시한다. Date → 날짜, Stock → 종목, ClosePrice → 일별 가격.
3. 사용자가 매핑을 확인하고 분석 목적으로 "성과 및 리스크 전체 분석"을 선택한다.
4. 시스템이 데이터 충분성 검사를 수행하고 다음 결과를 표시한다.
   - 가능한 분석: 성과 분석(누적 수익률, 기간 수익률), 드로다운 분석, 비중 분석 불가(비중 데이터 없음)
   - 제한적 가능 분석: 리스크 분석 — 변동성은 계산 가능하나 데이터 기간이 20거래일로 짧아 통계적 신뢰성이 낮음. Skills.md 규칙(최소 60거래일 권장)에 따라 경고 표시.
   - 불가능한 분석: 샤프 지수(무위험 수익률 미포함), 벤치마크 비교(벤치마크 데이터 없음), 집중도 분석(비중 데이터 없음)
5. 시스템이 선택지를 제시한다. 가능한 분석만으로 대시보드 생성, 또는 추가 데이터 업로드.
6. 사용자가 "가능한 분석만으로 진행"을 선택한다.
7. 대시보드가 생성된다. 비중 분포 패널에는 "보유 비중 데이터 없음 — holdings 또는 weight 컬럼 포함 파일 추가 시 분석 가능"이라는 안내가 표시된다. 리스크 패널에는 데이터 기간 짧음 경고가 포함된다.

### 시나리오 3: 컬럼 매핑 불확실 흐름 — 저신뢰도 컬럼 존재

**상황:** 페르소나 A가 사내에서 만든 비표준 형식의 Excel을 업로드한다. 컬럼명이 "수익률_조정", "날짜(서울시간)", "자산분류1"처럼 불명확하다.

1. 사용자가 XLSX 파일을 업로드한다.
2. AI가 컬럼 추론을 수행한다. 날짜(서울시간) → 날짜(신뢰도 0.85, 고신뢰). 수익률_조정 → 일별 수익률(신뢰도 0.62, 중간 신뢰). 자산분류1 → 자산군(신뢰도 0.41, 저신뢰).
3. 시스템이 매핑 결과를 신뢰도 구간별로 구분하여 표시한다.
   - 고신뢰 항목(자동 매핑, 사용자 확인): 날짜(서울시간) → 날짜
   - 중간 신뢰 항목(사용자 확인 요청): 수익률_조정이 "일별 수익률"에 매핑되었습니다. 확인하시겠습니까?
   - 저신뢰 항목(사용자 선택 필요): 자산분류1의 의미를 선택해주세요. 후보: 자산군, 업종, 전략 태그, 기타
4. 사용자가 중간 신뢰 항목을 확인하고 저신뢰 항목에서 "자산군"을 선택한다.
5. 매핑이 확정되고 이후 분석 흐름이 정상 진행된다.
6. 사용자가 확정한 매핑 결과가 Skills 로그에 기록된다. 재분석 시 같은 파일을 사용하면 동일한 매핑 확정 기록을 참조할 수 있다.

### 시나리오 4: 벤치마크 데이터 없는 흐름 — 벤치마크 비교 요청 *(P1 시나리오)*

> **분류:** 이 시나리오는 P1 기능(벤치마크 비교)을 포함하는 확장 사용 흐름이다. P0 데모 경로에 포함되지 않는다.

**상황:** 페르소나 B가 포트폴리오 수익률 CSV를 업로드하고 "KOSPI 대비 초과수익을 분석해줘"라고 입력한다.

1. 사용자가 파일을 업로드하고 분석 목적으로 "벤치마크 비교"를 포함한 자연어를 입력한다.
2. 시스템이 파일 내에서 벤치마크 수익률 또는 가격 컬럼을 탐색한다.
3. 벤치마크 데이터가 파일에 존재하지 않음을 감지한다.
4. 시스템이 다음을 안내한다. "벤치마크 비교에는 벤치마크 수익률 또는 가격 데이터가 필요합니다. 현재 업로드된 파일에서 벤치마크 데이터를 찾을 수 없습니다. KOSPI 지수 수익률을 별도 CSV로 추가 업로드하거나, 벤치마크 비교를 제외하고 진행하실 수 있습니다."
5. 사용자가 "벤치마크 비교 제외하고 진행"을 선택한다.
6. 성과 분석, 드로다운 분석, 리스크 분석이 수행되고 대시보드가 생성된다. 벤치마크 비교 패널은 "데이터 부족으로 표시 불가" 상태로 대시보드에 포함된다.
7. 이후 사용자가 벤치마크 CSV를 추가 업로드하면 벤치마크 비교 분석이 활성화되고 대시보드가 갱신된다.

## 섹션 8. AI 컬럼 매핑 정책

### 8.1 신뢰도 구간 정의 및 시스템 동작

| 신뢰도 구간 | 조건 | 시스템 동작 | 사용자 개입 |
|-----------|------|-----------|-----------|
| 고신뢰도 | 신뢰도 ≥ 0.8 | 자동 매핑 수행. 확인 화면에서 매핑 결과를 표시하고 일괄 확인 버튼 제공 | 선택적 (검토 후 확인 또는 개별 수정 가능) |
| 중간 신뢰도 | 0.5 ≤ 신뢰도 < 0.8 | 매핑 결과를 제안하되 개별 컬럼에 대해 사용자 확인 요청 | 필수 (개별 확인 없이 분석 진행 불가) |
| 저신뢰도 | 신뢰도 < 0.5 | 후보 의미 범주 목록을 제시하고 사용자 선택 요청 | 필수 (사용자 선택 없이 분석 진행 불가) |
| 충돌 | 두 개 이상의 컬럼이 동일한 의미 범주에 매핑된 경우 | 분석 차단. 충돌 컬럼과 충돌 역할을 표시하고 사용자에게 우선 컬럼 선택 요청 | 필수 (충돌 해소 없이 분석 진행 불가) |

### 8.2 매핑 결과 기록

- 확정된 매핑 결과는 Skills 로그에 기록된다.
- 기록 항목: 원본 컬럼명, 매핑된 의미 범주, 신뢰도 점수, 사용자 확인 여부(자동/수동), 확정 시각.
- 재현 목적: 동일 파일을 재업로드 시 이전 확정 매핑 기록을 참조할 수 있도록 세션 내에서 유지한다.

### 8.3 사용자가 확인을 건너뛸 경우

- 중간 신뢰도 이하 컬럼에 대한 확인을 건너뛰거나 닫으면 해당 컬럼이 포함된 분석은 차단된다.
- 시스템 메시지: "다음 컬럼의 의미가 확정되지 않았습니다: [컬럼명 목록]. 이 컬럼을 사용하는 분석은 수행할 수 없습니다. 확인 후 진행하거나, 해당 분석을 건너뛰시겠습니까?"

## 섹션 9. 재현성 및 검증 정책

### 9.1 결정적 출력 (deterministic)

다음 항목은 동일한 입력 파일, 동일한 확정 컬럼 매핑, 동일한 Skills.md 규칙 버전이 주어지면 동일한 결과를 생성한다.

| 결정적 항목 | 설명 |
|-----------|------|
| 수치 지표 계산 | MDD, 연환산 변동성, 누적 수익률, CAGR (데이터 기간 1년 이상인 경우에만 산출), 샤프 지수 (P1), HHI (P1), 추적오차 (P1), 정보비율 (P1) |
| 대시보드 레이아웃 구성 | 동일 가능 분석 항목 목록에서 Skills.md 레이아웃 규칙에 따라 동일 배치 생성 |
| 차트 유형 선택 | 동일 데이터 구조에서 Skills.md 시각화 규칙에 따라 동일 차트 유형 선택 |
| 데이터 충분성 판단 결과 | 동일 컬럼 매핑과 데이터 기간에서 동일 가능/불가 판단 출력 |

### 9.2 비결정적 출력 (non-deterministic)

다음 항목은 AI 보조 생성이므로 동일 입력에서도 출력이 달라질 수 있다. 사용자 인터페이스에서 비결정적 출력임을 명시한다.

| 비결정적 항목 | 설명 |
|-----------|------|
| 컬럼 의미 추론 초기 결과 | AI가 제안하는 컬럼 매핑 후보 및 신뢰도 점수. 사용자 확인으로 결정적 상태가 된다. |
| 애널리스트 패널 해석 텍스트 | 수치 지표 기반으로 생성되는 자연어 해석. 내용은 달라질 수 있으나 참조한 수치 지표는 동일하다. |

### 9.3 Skills.md의 분석 제어 역할

각 분석 단계는 Skills.md의 특정 규칙을 참조한다. 규칙은 다음 유형으로 구성된다.

| 규칙 유형 | Skills.md 내 정의 내용 | 예시 |
|---------|---------------------|------|
| 입력 스키마 규칙 | 분석에 필요한 컬럼 의미 범주 및 필수/선택 구분 | 성과 분석 필수: 날짜, 일별 수익률 또는 일별 가격 |
| 지표 계산 규칙 | 수식 정의, 사용 단위, 연환산 방법 | 연환산 변동성 = 일별 수익률 표준편차 × √252 |
| 데이터 충분성 규칙 | 최소 기간, 최소 종목 수, 결측치 허용 한도 | 리스크 분석 권장 최소 60거래일 |
| 시각화 선택 규칙 | 데이터 구조별 차트 유형 기준 | 수익률 시계열 → 선 차트, 비중 분포 → 파이차트 (트리맵은 P1 확장 기능) |
| 위험 임계값 규칙 | 경고를 트리거하는 지표 임계값 | MDD ≤ -20%이면 고위험 경고 표시 |
| 인사이트 생성 규칙 | 리포트 섹션 구성 순서, 포함 항목 정의 | 요약 → 성과 결과 → 리스크 결과 → 위험 요인 → 데이터 한계 |

| 전사 BI 플랫폼 기능 | 범용 데이터 파이프라인, 조직 권한 관리, 대용량 데이터 처리는 이 서비스의 목적이 아님 |

## 섹션 13. 다음 Stage로 넘길 항목

| Feature Spec 문서 (Stage 7) | 섹션 6의 각 Feature에 대해 수용 기준을 테스트 케이스 형식으로 확장. 입력/기대 출력/판정 기준으로 구성된 Feature별 검증 케이스 작성. F-CM-01~03 컬럼 매핑 정확도 검증 케이스, F-DS-01~03 데이터 충분성 판단 검증 케이스, F-AN-01~06 수치 지표 계산 검증 케이스 포함. |
