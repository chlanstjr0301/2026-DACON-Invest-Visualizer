## 1. 공모전 핵심 요구사항 해석

### 1.1 대회 공식 명칭이 암시하는 것

"월간 해커톤: 투자 데이터를 시각화하라 — Skills 기반 대시보드 설계"

이 명칭에서 주목해야 할 단어는 세 개다.

| 단어 | 표면적 의미 | 우리의 해석 |
|------|-----------|------------|
| 투자 데이터 | 금융 데이터 일반 | 포트폴리오·수익률·거래내역 등 구조가 다양한 투자 데이터 |
| 시각화 | 차트 생성 | 단순 차트가 아니라 분석 목적에 따라 적절히 선택된 시각화 |
| Skills 기반 | Skills.md 문서 활용 | 분석 규칙을 문서화하고 그 규칙에 근거해 대시보드를 생성하는 구조 |

### 1.2 핵심 요구사항의 본질 (해석)

공모전이 실제로 요구하는 것은 다음 한 가지라고 판단한다.

> "분석 규칙을 문서화할 수 있는가, 그 규칙이 실제 대시보드 생성에 작동하는가."

화려한 UI나 기능의 수보다 **규칙-결과 간의 추적 가능한 연결**이 핵심이다. Skills.md는 단순한 설명 문서가 아니라, 대시보드가 어떤 판단을 하는지를 정의하는 명세서 역할을 해야 한다고 본다.

### 1.3 "바이브코딩 활용" 항목의 해석

배점표에 "바이브코딩 활용" 항목이 15점으로 명시되어 있다. 공식 평가 내용은 "문서 기반 생성 구조, 자동 생성 수준, 수동 구현 최소화"다.

우리의 해석:
- Skills.md가 AI 코드 생성의 기준 문서로 실제 작동해야 한다.
- 단순히 "LLM을 사용했다"는 사실이 아니라, **Skills.md → 코드 생성 → 대시보드 결과**의 연결 구조를 보여야 한다.
- 수동 코딩 비중이 낮고, 그 근거를 문서로 설명할 수 있어야 한다.

## 2. 명시적 요구사항

*※ 이하는 contest_brief.md에 명시된 공식 요건이다.*

### 2.1 제출물 목록

| 제출물 | 형식 | 포함 내용 |
|--------|------|----------|
| 기획서 | PDF | 서비스 개요, 분석 흐름 설계, 대시보드 구성, Skills.md 설계 방향, 확장 기능 아이디어 |
| Skills.md 문서 | .md (PDF 변환 제출) | 분석 규칙, 시각화 기준, 대시보드 구성 규칙. 1개 또는 복수 제출 가능 |
| 웹 페이지 | 배포 URL (필수), Github 링크 (선택) | Skills.md 기반 바이브코딩으로 구현한 투자 대시보드 웹 서비스 |

### 2.2 평가 구조

**1차 평가: 대중 투표**

| 투표 주체 | 가중치 |
|----------|--------|
| 제출팀 투표 | 60% |
| 참가팀 투표 | 20% |
| 대중 투표 | 20% |

상위 10팀이 2차 평가로 진출한다. 동점 시 참가팀 투표 점수 높은 팀, 이후 업로드 시간이 빠른 팀 우선.

**2차 평가: 내부 심사위원 정성 평가 (100점 만점)**

| 평가항목 | 배점 | 공식 평가 내용 |
|---------|-----|--------------|
| 범용성 | 25 | 투자 데이터 구조 대응 가능성, 재사용 가능성 |
| Skills.md 설계 | 25 | 분석 규칙 명확성, 시각화 기준, 인사이트 생성 규칙 |
| 대시보드 자동 생성 | 25 | 자동 분석 동작, 시각화 적절성, UI 구성 완성도 |
| 바이브코딩 활용 | 15 | 문서 기반 생성 구조, 자동 생성 수준, 수동 구현 최소화 |
| 실용성 및 창의성 | 10 | 실제 활용 가능성, 확장 기능, 사용자 경험 개선 |

**관찰:** 상위 3개 항목(범용성, Skills.md 설계, 대시보드 자동 생성)이 각각 25점으로 합산 75점을 차지한다. 실용성·창의성은 10점이다. 구조적 완성도가 창의성보다 배점상 훨씬 중요하다.

### 2.3 기술 제약

- 배포 URL은 심사 기간 동안 외부 접근 가능해야 한다.
- 기술 선택은 자유다.
- 더미 데이터 또는 공개 데이터 활용 가능하다.
- 외부 API 사용 시 심사자가 별도 키 없이 확인 가능해야 한다.
- 생성형 AI 도구 사용은 허용된다.

## 3. 암묵적 평가 기준

*※ 이하는 공식 평가 기준이 아닌 전략적 해석이다. 심사위원이 평가 기준 외에 실질적으로 판단할 것으로 예상되는 항목을 정리한다.*

### 3.1 Skills.md가 실제로 분석을 제어하는가

단순히 Skills.md 문서를 잘 작성했느냐가 아니라, 그 문서에 적힌 규칙이 실제 대시보드 출력에 연결되어 있음을 보여야 한다. 예를 들어, Skills.md에 "MDD가 -20% 이하이면 고위험 경고를 표시한다"는 규칙이 있다면, 대시보드에서 그 경고가 실제로 나타나야 한다. 문서와 동작이 분리된 기획서는 감점 요인으로 예상된다.

### 3.2 데이터 불충분 상황을 어떻게 처리하는가

사용자가 업로드한 데이터가 요청한 분석에 충분하지 않을 때 시스템이 어떻게 반응하는지가 설계 성숙도를 보여준다. "분석 불가"를 출력하는 것과, 어떤 분석은 가능하고 어떤 분석은 추가 데이터가 필요하다고 구체적으로 안내하는 것은 평가 차이가 클 것으로 판단한다.

### 3.3 분석 결과의 재현성과 감사 가능성

같은 데이터를 넣었을 때 항상 같은 결과가 나오는지, 그리고 왜 그 결과가 나왔는지 추적할 수 있는지가 중요하다. 금융 도메인에서 재현 불가능한 분석 결과는 신뢰받기 어렵다는 점에서, 이것이 규칙 기반 분석 설계의 핵심 근거가 된다.

### 3.4 범용성과 금융 특화 사이의 균형

범용성 항목이 25점이지만, 범용성을 너무 넓게 잡으면 기존 범용 BI·데이터 분석 도구와 구분이 없어진다. 심사위원은 "임의의 투자 CSV를 처리할 수 있는 유연함"을 원할 것으로 보이며, "모든 종류의 데이터를 처리한다"는 과장은 오히려 실현 가능성 의심을 유발할 수 있다. 포트폴리오 수준 투자 데이터에 집중하면서, 컬럼 구조가 다양해도 처리할 수 있다는 것이 올바른 범용성이다.

### 3.5 대중 투표 전략과 첫 화면 설계의 연결

1차 평가는 대중 투표(제출팀 60%, 참가팀 20%, 대중 20%)다. 기획서 PDF와 웹 서비스의 첫 인상이 득표에 직접 영향을 준다.

**첫 화면이 갖춰야 할 조건:**
- 서비스가 무엇을 하는지 3초 안에 전달해야 한다.
- 데모 시나리오가 즉시 시작될 수 있어야 한다 (CSV 샘플 파일 제공 또는 원클릭 데모).
- 대시보드 결과물이 첫 화면에서 보여야 하며, 분석 규칙이 어디서 왔는지 한 줄이라도 표시되어야 한다.
- 복잡한 기술 설명은 첫 화면에서 배제하고, 두 번째 뎁스에 배치한다.

대중 투표에서 우위를 점하려면 기획서의 첫 2페이지와 랜딩 페이지가 MVP 데모 시나리오와 직접 연결되어야 한다 (섹션 8 참조).

| 조건 | 이유 |
|------|------|
| Skills.md 규칙과 대시보드 출력 간의 명확한 연결 | 공모전의 핵심 평가 대상이 Skills.md 설계 (25점)이기 때문 |
| 데이터 구조 다양성에 대한 명시적 처리 방식 | 범용성 항목 (25점)이 요구하는 핵심 능력 |

### 4.2 심사위원이 감점할 것으로 예상되는 것

| 위험 요소 | 이유 |
|----------|------|
| "AI가 모든 투자 데이터를 분석한다"는 과장 | 범위가 불명확하면 실제 구현 신뢰성이 낮아 보인다 |
| Skills.md가 단순 설명 문서에 그치는 경우 | 규칙-결과 연결이 없으면 Skills.md 설계 항목에서 낮은 점수를 받는다 |

| 기존 서비스와의 차별화 논리가 없는 경우 | 공모전 목적에 맞지 않는 진부한 제안으로 보일 수 있다 |
| 실시간 시세·자동매매 포함 시도 | MVP 실현 가능성을 낮추고 심사 기간 중 기능 오작동 위험이 있다 |
| 바이브코딩을 "AI 도구 사용"으로만 설명하는 경우 | 문서 기반 생성 구조가 없으면 배점 기준을 충족하지 못한다 |

## 5. 우리 서비스가 잡아야 할 포지션

### 5.1 포지션 정의

Skillfolio는 다음과 같이 정의된다.

> 사용자가 업로드한 비정형 투자 CSV/XLSX 파일을 Skills.md에 정의된 금융 분석 규칙으로 처리하여, 재현 가능한 포트폴리오 분석 대시보드와 애널리스트 형식의 리포트를 자동 생성하는 서비스.

이 정의는 세 축으로 구성된다.

| 축 | 내용 |
|----|------|
| 입력 유연성 | 컬럼 구조가 다른 CSV/XLSX도 AI 컬럼 매핑으로 처리한다 |
| 규칙 기반 분석 | 분석은 Skills.md에 명시된 규칙을 따르며, 결과는 재현 가능하다 |
| 결과 형식 | 채팅 답변이 아니라 대시보드와 리포트로 출력된다 |

### 5.2 포지션 경계 (무엇이 아닌가)

*※ 이하는 공개된 제품 설명을 기반으로 한 기획 해석이며, 각 서비스의 공식 한계 선언이 아니다.*

| 아닌 것 | 우리의 해석 |
|---------|-----------|
| 범용 BI 도구 | Power BI, Tableau는 범용 BI 플랫폼으로, 금융 포트폴리오 전용 분석 규칙을 내장하지 않는다. Skillfolio는 포트폴리오 분석으로 범위를 한정한다 |
| 범용 AI 데이터 분석기 | Julius AI는 범용 데이터 분석 도구로 일반적인 금융 질문에 답하는 방식으로 설계되어 있다. Skillfolio는 Skills.md 규칙이 분석 흐름을 제어하는 방식을 취한다 |
| 거래 자동화 서비스 | 자동매매·주문 실행 기능은 Skillfolio의 범위 밖이다. 분석과 리포트 생성에서 멈춘다 |
| 실시간 시세 플랫폼 | Skillfolio는 외부 시장 데이터 연결 없이 사용자 업로드 파일만 분석한다 |
| 세금·배당 관리 도구 | 세금 리포트, 배당 재투자 추적은 범위 외다. 성과·리스크·비중·집중도 분석에 집중한다 |

### 5.3 벤치마크 gap 분석 기반 포지션 근거

*※ 이하는 공개된 제품 설명 및 web_research_notes.md 검토를 기반으로 한 기획 해석이다.*

| 경쟁 서비스 유형 | 강점 (공개 정보 기반) | 우리가 채우려는 빈틈 (해석) |
|---------------|---------------------|--------------------------|
| Portfolio Visualizer, Koyfin 등 금융 특화 서비스 | 높은 분석 품질과 지표 신뢰성 | 사용자 고유의 비정형 CSV 파일을 직접 받아 처리하는 시나리오가 주된 사용 흐름으로 제시되지 않음 |
| Power BI Copilot, Julius AI 등 범용 AI 분석 도구 | 자연어 UX, 파일 업로드 지원 | 금융 포트폴리오 전용 분석 규칙이 내장되지 않으며, Microsoft 공식 문서는 Copilot 생성 보고서가 비결정적임을 명시함 |
| 전체 범주 공통 | 각자의 영역에서 완성도 높음 | 금융 특화 분석 규칙 + 유연한 CSV 입력 + 재현 가능한 대시보드를 동시에 제공하는 서비스 구조가 명확히 보이지 않음 |

Skillfolio는 위 세 가지 빈틈을 동시에 채우는 구조를 목표로 한다. 이 해석이 공모전의 "범용성", "Skills.md 설계", "대시보드 자동 생성" 배점 항목과 직접 대응한다.

## 6. 피해야 할 방향

### 6.1 과장된 범위 선언

"모든 금융 데이터를 분석한다"거나 "AI가 투자 의사결정을 지원한다"는 표현은 삼가야 한다. 심사위원에게 구현 가능성에 대한 의심을 유발하고, 실제 MVP와 기획서의 괴리를 드러낸다.

대신: 포트폴리오 수준의 CSV/XLSX 분석, 성과·리스크·비중·집중도 분석으로 범위를 명확히 한다.

### 6.2 Skills.md를 단순 프롬프트로 취급하는 것

Skills.md를 "AI에게 지시하는 프롬프트" 수준으로 작성하면 안 된다. 입력 스키마 규칙, 지표 계산 규칙, 시각화 선택 기준, 인사이트 생성 규칙, 리포트 구성 흐름이 구체적으로 정의된 분석 명세서여야 한다.

### 6.3 데이터 불충분 상황의 무처리

분석에 필요한 컬럼이 없거나 데이터 품질이 낮을 때 시스템이 침묵하거나 오류를 내는 설계는 평가 감점 대상이다. 어떤 분석은 가능하고 어떤 분석은 불가능하며 어떤 추가 데이터가 필요한지를 사용자에게 명확히 전달하는 설계가 필요하다.

### 6.4 MVP 범위 과확장

실시간 시세, 자동매매, 세금 계산, 틱 데이터, 팩터 리서치 플랫폼 기능을 기획서에 포함하는 것은 위험하다. 핵심인 Skills.md 기반 분석 품질에서 집중력을 분산시킨다.

### 6.5 바이브코딩을 형식적으로만 언급하는 것

"Claude를 사용하여 코드를 생성했다"는 설명만으로는 바이브코딩 활용 배점을 충족하지 못한다. Skills.md가 구체적으로 어떤 대시보드 컴포넌트 생성의 기준이 되었는지를 보여야 한다.

### 6.6 단일 종목 중심 설계

단일 종목 대시보드는 일반 증권사 앱과 차별화가 없다. 최소 2개 이상의 종목 또는 포트폴리오 전체를 대상으로 하는 분석 구조가 필요하다.

## 7. 최종 제품 thesis

우리가 제안하는 서비스는 범용 대시보드도 범용 챗봇도 아니다.

Skillfolio는 투자 전문가가 보유한 비정형 CSV/XLSX 투자 데이터를 Skills.md에 정의된 포트폴리오 분석 규칙으로 처리하여, 재현 가능한 성과·리스크·비중·집중도 분석 대시보드와 애널리스트 형식의 리포트를 자동 생성하는 서비스다.

기존 금융 분석 서비스는 고정된 입력 스키마를 전제로 설계되어 있고, 기존 범용 AI 데이터 분석 도구는 금융 포트폴리오 전용 분석 규칙을 내장하지 않는다. Skillfolio는 Skills.md 기반의 규칙 문서와 AI 컬럼 매핑을 통해 이 두 가지 한계를 동시에 다루며, 그 설계 구조가 공모전의 핵심 평가 항목인 범용성·Skills.md 설계·대시보드 자동 생성과 직접 대응한다.

## 8. MVP 데모 시나리오

*※ 대중 투표 전략과 첫 화면 설계를 연결하기 위한 참조 시나리오. 세부 구현 방식은 이후 단계에서 확정한다.*

**시나리오: "5종목 포트폴리오 CSV 업로드 → 대시보드 자동 생성"**

| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1. 파일 업로드 | 5개 종목, 1년치 일별 수익률이 담긴 CSV를 드래그 앤 드롭 | 파일 수신, 컬럼 목록 표시 |
| 2. 컬럼 매핑 확인 | AI가 제안한 컬럼 매핑(날짜, 종목명, 수익률)을 확인 또는 수정 | 매핑 결과를 표로 표시, 수정 가능 |
| 3. 분석 목표 선택 | "포트폴리오 성과 및 리스크 분석" 선택 | 데이터 충분성 판단 결과 표시 |
| 4. 데이터 충분성 확인 | 충분성 결과 확인: "성과·MDD 분석 가능, 벤치마크 비교 불가 (벤치마크 데이터 없음)" | 가능한 분석과 불가 분석 목록 표시 |
| 5. 대시보드 생성 | "대시보드 생성" 클릭 | Skills.md 규칙 기반 대시보드 자동 생성: 누적 수익률 차트, MDD 차트, 비중 파이차트, 변동성 바차트 |
| 6. 리포트 확인 | 대시보드 옆 패널에서 인사이트 텍스트 확인 | 각 지표의 해석을 애널리스트 형식으로 표시. Skills 로그 열람 가능 |
| 7. 내보내기 | PDF 또는 PNG 내보내기 | 리포트 파일 다운로드 |

**랜딩 페이지 연결 원칙:**
- 이 시나리오를 그대로 랜딩 페이지의 첫 화면 데모로 사용한다.
- 샘플 CSV를 제공하여 방문자가 즉시 업로드 경험을 할 수 있게 한다.
- 대시보드 결과 스크린샷이 랜딩 페이지 상단 절반을 차지해야 한다.
- "이 대시보드는 Skills.md 규칙으로 생성되었습니다"라는 한 줄 레이블을 대시보드 하단에 배치한다.
