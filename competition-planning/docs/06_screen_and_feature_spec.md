### 전체 레이아웃 (P0 대시보드 기준)

```
┌─────────────────────────────────────────────────────────────────┐
│  상태 바: 파일명 | 매핑 상태 | 분석 목적 | 단계 진행 표시         │
├──────────────────────────────────┬──────────────────────────────┤
│                                  │                              │
│  대시보드 캔버스 (좌측)            │  AI 애널리스트 패널 (우측)    │
│  - 누적 수익률 선 차트             │  - Skills.md 템플릿 해석문    │
│  - 구간 수익률 테이블              │  - 데이터 한계 고지           │
│  - MDD 수치 + 드로다운 차트        │  - 위험 판정 레이블           │
│  - 연환산 변동성 테이블            │                              │
│  - 기본 비중 차트 (비중 컬럼 존재시) │                              │
│                                  │                              │
├──────────────────────────────────┴──────────────────────────────┤
│  Skills 로그 슬라이드오버 (하단 토글 또는 슬라이드인)              │
│  [Column Mapping] [Sufficiency] [Metric] [Visualization] [Insight]│
└─────────────────────────────────────────────────────────────────┘
```

**설계 원칙:**
- 좌측 = 사실 (수치·차트), 우측 = 해석 (AI 애널리스트 패널)
- Skills 로그는 항상 접근 가능하며, 대시보드 카드 하단에 참조 규칙 ID 표시
- "데이터 부족" 패널은 공백이 아닌 사유와 필요 데이터를 명시한 상태 패널로 표시
- P0 흐름은 단일 경로: 업로드 → 매핑 → 목적 → 충분성 → 대시보드 → 로그 확인

## 섹션 4. 핵심 화면 설명

**SCR-01 파일 업로드**
- 목적: 비정형 CSV/XLSX를 수신하고 형식 검증
- 표시 요소: 드래그앤드롭 영역, 지원 형식 안내(CSV/XLSX), 파일 크기 한도(10MB)
- 사용자 행동: 파일 드롭 또는 선택
- 시스템 반응: 확장자·인코딩·크기 검증 → 통과 시 컬럼 목록+미리보기 → SCR-02 이동 / 실패 시 오류 메시지

**SCR-02 데이터 미리보기 & 컬럼 매핑**
- 목적: AI 추론 결과를 사용자가 확인·수정·확정
- 표시 요소: 상위 5행 미리보기 테이블, 컬럼명-의미 범주-신뢰도 매핑 테이블
- 사용자 행동: 고신뢰도 항목 일괄 확인, 중간 신뢰도 항목 개별 팝업 확인, 저신뢰도 항목 드롭다운 수동 선택
- 시스템 반응: 확정된 매핑을 Column Mapping Log에 기록 → SCR-03 이동

**SCR-03 분석 목적 입력**
- 목적: 사용자 분석 의도를 확정하고 모호성을 명확화
- 표시 요소: 자연어 입력창, 목적 후보 카드(성과 분석/위험 분석/비중 분석), 자동 진단 선택지
- 사용자 행동: 자연어 입력 또는 후보 카드 선택
- 시스템 반응: 모호한 입력 시 목적 후보 팝업 제시 → 확정 후 SCR-04 이동

**SCR-04 데이터 충분성 결과**
- 목적: 분석 가능 여부를 항목별로 명시하여 사용자가 진행 또는 보완을 선택
- 표시 요소: 분석 항목-판정 결과-사유-필요 데이터 테이블, "데이터 부족" 항목 강조
- 사용자 행동: 가능한 분석만으로 진행 확정 또는 추가 데이터 보완
- 시스템 반응: Sufficiency Check Log 기록 → 진행 확정 시 SCR-05 이동

**SCR-05 대시보드 메인**
- 목적: Skills.md 규칙 기반 P0 핵심 분석 결과를 시각화
- 표시 요소: 누적 수익률 선 차트, 구간 수익률 테이블, MDD 수치+드로다운 차트, 연환산 변동성 테이블, 기본 비중 차트(비중 컬럼 존재 시), 각 카드 하단 참조 규칙 ID
- 사용자 행동: 차트 열람, AI 패널 확인, Skills 로그 열기
- 시스템 반응: 대시보드 카드 렌더링, 우측 AI 패널 자동 표시, 로그 패널 접근 가능

**SCR-06 AI 애널리스트 패널**
- 목적: Skills.md 템플릿 기반 해석문을 표시하고 AI 자연어 서술로 보조
- 표시 요소: 템플릿 기반 해석문(위험 판정 레이블 포함), 데이터 한계 고지, AI 보조 서술
- 사용자 행동: 해석문 열람
- 시스템 반응: 금지 문구(투자 권유, 미래 수익 전망, 인과 단정) 없음 검증 후 표시

**SCR-07 Skills 로그 패널**
- 목적: 모든 분석 단계의 규칙 ID·입출력·판정 근거를 열람
- 표시 요소: 5개 유형 탭(Column Mapping / Sufficiency Check / Metric Calculation / Visualization Rule / Insight Rule), 탭별 필드 상세
- 사용자 행동: 탭 전환, 항목별 규칙 ID 확인
- 시스템 반응: 해당 대시보드 카드로 포커스 이동(양방향 연결)

**SCR-08 P1/P2 확장 안내**
- 목적: P0 경계 외 기능 요청 시 P1/P2 확장 범위를 안내
- 표시 요소: 요청된 기능명, P1/P2 분류, 필요 추가 데이터, P0 데모 외 범위 명시 배너
- 사용자 행동: 안내 확인 후 SCR-05로 복귀
- 시스템 반응: P0 분석 결과는 유지, 확장 기능은 별도 안내만 표시

## 섹션 6. P1/P2 경계 표

### P1 기능 (MVP 이후 1차 확장)

| 기능 | 설명 | 진입 조건 |
|------|------|---------|
| Holdings/비중 독립 분석 | 보유 종목·비중 파일만으로 집중도·섹터 분석 | Holdings 파일 단독 업로드 |
| 벤치마크 비교 | 포트폴리오 vs 벤치마크 누적 수익률 비교 + 초과 수익 | 파일 내 벤치마크 수익률 컬럼 존재 |

| 다중 파일 업로드 | 복수 파일 결합 분석 | P0 단일 파일 이후 |

### P2 기능 (장기 확장)

| 기능 | 설명 |
|------|------|
| PDF/PNG 내보내기 | 대시보드 화면을 고해상도 파일로 내보내기 |

| 고급 벤치마크 지표 | 트래킹 에러, 정보 비율, 알파/베타 |
| 대시보드 레이아웃 커스터마이징 | 카드 순서·크기 수동 편집 |

> **주의:** 거래 내역 기반 성과 계산(수익률·MDD·변동성 역산)은 현금 흐름·수수료 처리 복잡도로 P1+ 이상 유지. P1에서는 거래 목록 표시 및 기본 요약에 한정.

## 섹션 7. 데모 화면 시나리오 (P0 핵심 경로)

P0 데모는 페르소나 A의 월간 성과 보고 자동화 흐름을 시연한다. 5종목 1년치 일별 수익률 CSV를 사용한다.

| 단계 | 화면 | 핵심 장면 | 심사위원 주목 포인트 |
|------|------|---------|-----------------|
| 1 | SCR-01 | 비정형 CSV 파일 드래그 앤 드롭 | "어떤 CSV든 업로드 가능" |
| 2 | SCR-02 | AI가 컬럼명을 자동 추론 + 신뢰도 표시 | 비정형 입력의 자동 처리 |
| 3 | SCR-02 | 사용자가 한 번 클릭으로 매핑 확정 | 수작업 없는 진입 |
| 4 | SCR-03 | "이 포트폴리오 성과 분석해줘" 입력 → 후보 카드 제시 → 선택 | 목적 기반 라우팅 |
| 5 | SCR-04 | 분석 항목별 판정 결과 표시 (가능/불가 명시) | 데이터 충분성 검증 |
| 6 | SCR-05 | 누적 수익률·MDD·변동성·비중 대시보드 자동 생성 | Skills.md 기반 자동화 |
| 7 | SCR-06 | AI 애널리스트 패널: 위험 레이블 + 해석문 | 규칙 기반 해석, 투자 권유 없음 |
| 8 | SCR-07 | Skills 로그 팝업: 5개 유형 탭, 규칙 ID 확인 | 재현성 증거물 |

**10초 오프닝 장면:** 파일 드롭(2초) → 매핑 확인(3초) → 대시보드 생성(3초) → Skills 로그 팝업(2초)

## 섹션 8. Stage 8 인계 사항

Stage 8(Skills.md 설계)에서 다음 항목을 반드시 정의해야 한다.

| 인계 항목 | 내용 | 해당 화면 |
|---------|------|---------|
| 규칙 ID 체계 | RULE-CM-001 (컬럼 매핑), RULE-DS-001 (충분성 검사), RULE-AN-001~N (지표 계산), RULE-VZ-001~N (시각화), RULE-IN-001~N (인사이트) 형식 확정 | SCR-02~07 전체 |
| 규칙 스키마 | 각 규칙 ID의 필드 구조: rule_id, rule_type, description, input_spec, output_spec, threshold, formula | SCR-07 Skills 로그 |
| 지표 계산 공식 | 누적 수익률·구간 수익률·MDD·드로다운·연환산 변동성의 공식 및 거래일수 기준 명세 | SCR-05, F-AN-01~04 |
| 시각화 선택 규칙 | 데이터 유형별 차트 유형 결정 로직: 시계열 → 선 차트, 비중 → 파이/바 차트, 드로다운 → 영역 차트 | SCR-05, F-DB-01 |
| 인사이트 템플릿 | MDD 임계값(예: -20% 경고), 변동성 임계값, 위험 레이블 결정 기준, 해석문 템플릿 구조 | SCR-06, F-AI-01 |
| Skills 로그 스키마 | 5개 유형별 필드 명세: Column Mapping Log (원본명, 범주, 신뢰도, 확인 여부, 확정 시각), Sufficiency Check Log (항목명, 판정, 부족 데이터, 규칙 ID, excluded_analysis_reason), Metric Calculation Log (지표명, calculation_formula, rule_input, rule_output, 규칙 ID), Visualization Rule Log (분석 유형, 차트 유형, chart_selection_reason, 규칙 ID), Insight Rule Log (참조 지표, 임계값 규칙, 위험 판정, AI 보조 여부) | SCR-07, F-SL-01 |
| 결정적/비결정적 경계 | 수치 계산(결정적): 동일 입력+매핑+규칙 → 동일 출력. AI 보조(비결정적): 해석문 자연어 표현. 두 경계를 Skills.md에 명시 | SCR-05, SCR-06 |
| CAGR 조건부 표시 규칙 | CAGR은 거래일수 ≥252일 충족 시에만 참고용으로 표시, 핵심 지표 아님, 경고 레이블 필수 | SCR-05 (조건부) |
| 재현성 계약 | 동일 조건에서 결정적 출력이 보장되는 항목 목록 및 비결정적 항목 명시 문서화 | SCR-05~07 전체 |
