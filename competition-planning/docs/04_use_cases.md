## 섹션 1. Stage 4 목적

이 문서는 Skillfolio의 Stage 4 산출물로, 이전 단계(전략 메모·문제 정의·제품 개념·PRD)와 Review A(Codex·Gemini)의 피드백을 통합하여 **사용자가 실제로 마주치는 구체적인 사용 흐름**을 Use Case 단위로 정의한다.

Stage 4의 핵심 역할은 다음과 같다.

첫째, PRD의 기능 요구사항(F-UP, F-CM, F-GC, F-DS, F-AN, F-DB, F-AI, F-SL)이 실제 사용 시나리오에서 어떤 순서로 조합되는지 명시한다.

둘째, P0·P1·P2 우선순위를 사용 시나리오 단위로 재확인하고, 각 Use Case에서 가능한 분석·제한적 가능 분석·불가능한 분석을 명확히 분리한다.

셋째, Skills.md 규칙이 분석 흐름의 어느 지점에서 개입하는지, Skills 로그가 어떤 항목을 기록하는지 서술한다.

넷째, Stage 5(User Flow)와 Stage 6(Screen Spec) 작성자가 이 문서를 참조하여 Mermaid 다이어그램 및 화면 명세를 작성할 수 있도록 충분한 흐름 정보를 제공한다.

> **핵심 원칙:** Skillfolio는 범용 대시보드도 범용 챗봇도 아니다. Skills.md에 정의된 금융 포트폴리오 분석 규칙이 분석 흐름 전체를 제어하고, AI 컬럼 매핑이 비정형 입력 파일의 진입 장벽을 제거하며, 모든 분석 결과가 Skills 로그로 추적 가능한 형태로 출력된다.

## 섹션 2. Use Case 설계 원칙

### 원칙 1. User = Data + Goal 원칙

사용자는 두 가지를 가지고 있다. 업로드한 파일(데이터)과 분석하고 싶은 것(목적)이다. 시스템은 이 두 가지를 연결하는 역할을 한다. 어느 한쪽이 불분명하면 시스템은 분석을 강행하지 않고 명확화 단계를 먼저 수행한다.

### 원칙 2. 데이터 충분성 우선 검사

목적이 확정된 후 분석을 실행하기 전에, 시스템은 반드시 다음 순서로 데이터 충분성을 검사한다.

1. 업로드된 파일 내에 해당 분석에 필요한 컬럼이 존재하는가
2. 존재하지 않는 경우, 대체 컬럼(예: 가격 컬럼에서 수익률 역산)으로 분석 가능한가
3. 대체도 불가한 경우, 부분 분석이 가능한가
4. 부분 분석도 불가한 경우, 어떤 추가 데이터가 필요한지 명시한다

이 검사를 통과하지 않은 분석 항목은 대시보드에 "데이터 부족" 상태 패널로 표시된다. 침묵하거나 오류를 내지 않는다.

### 원칙 3. 목적 모호성 명확화 우선

사용자가 "이 포트폴리오 괜찮은지 봐줘"처럼 모호한 목적을 입력하면, 시스템은 분석을 즉시 시작하지 않는다. AI 애널리스트 패널이 분석 목적 후보를 제시하고 사용자가 선택 또는 자동 진단을 선택하도록 유도한다. 사용자 확인 없이 목적을 일방적으로 결정하지 않는다.

### 원칙 4. 침묵 대신 부분 분석

데이터가 불충분하더라도 시스템은 침묵하지 않는다. 가능한 분석은 수행하고, 불가능한 분석은 이유와 함께 안내한다. 부분 대시보드가 전체 대시보드보다 낫고, 정직한 "데이터 부족" 패널이 공백보다 낫다.

### 원칙 5. Skills.md를 규칙 소스로

모든 금융 지표 계산, 시각화 선택, 위험 경고 임계값, 리포트 템플릿은 Skills.md에 사전 정의된 규칙에 따른다. AI가 임의로 계산 방식을 선택하거나 임계값을 설정하지 않는다. 계산 방식이 Skills.md에 정의되어 있어야만 해당 분석이 실행된다.

### 원칙 6. Skills 로그를 Skills.md 제어의 증거로

각 분석 결과는 Skills 로그에 기록된다. Skills 로그는 단순한 감사 기록이 아니라, "Skills.md가 실제로 대시보드 생성을 제어했다"는 공모전 제출 증거물이다. 로그는 5개 유형으로 구성된다.

- **Column Mapping Log**: 원본 컬럼명, 추론 의미 범주, 신뢰도, 사용자 확인 여부, 확정 시각
- **Sufficiency Check Log**: 분석 항목명, 판정 결과(가능/제한적 가능/불가), 부족 데이터, 참조 규칙 ID, 제외 사유
- **Metric Calculation Log**: 지표명, 계산 공식, 입력값(rule_input), 출력값(rule_output), 참조 규칙 ID
- **Visualization Rule Log**: 분석 유형, 선택된 차트 유형, 차트 선택 사유(chart_selection_reason), 참조 규칙 ID
- **Insight Rule Log**: 참조 수치 지표, 임계값 규칙, 위험 판정 결과, AI 보조 문장 여부

### 원칙 7. P0/P1/P2 우선순위 분리

이 문서의 모든 Use Case는 PRD 및 review_A_resolution.md에서 확정된 우선순위를 따른다. P0 Use Case는 MVP 데모에서 반드시 동작해야 한다. P1·P2 Use Case는 확장 기능으로, P0 데모에 포함되지 않는다.

**P0에서 명시적으로 제외된 기능 목록 (P1 이상):**
- 거래내역 기반 성과 계산
- 샤프 지수
- 집중도 분석 / HHI
- 벤치마크 비교
- 트리맵
- Markdown 파일 다운로드
- PDF/PNG 내보내기
- 다중 파일 결합

### UC-01. 일별 수익률 CSV 기반 포트폴리오 성과·리스크·비중 대시보드 자동 생성

| 항목 | 내용 |
|------|------|
| **우선순위** | P0 — MVP 대표 데모 경로 |
| **주요 사용자** | 페르소나 A (투자 운용 실무자) |
| **상황** | 페르소나 A는 5개 종목의 1년치 일별 수익률이 담긴 CSV를 보유하고 있다. 컬럼명은 비표준일 수 있다(예: "수익률_조정", "날짜(서울시간)"). 월간 성과 리포트 작성 전에 포트폴리오 전반을 신속하게 진단하고 싶다. |
| **사용자 목적** | 포트폴리오의 성과(누적 수익률, 기간 수익률), 리스크(MDD, 연환산 변동성), 비중 분포를 한 화면에서 확인하고 분석 근거를 추적할 수 있는 형태로 출력받는다. |
| **입력 데이터** | CSV 파일. 5개 종목, 약 252거래일(1년치) 일별 데이터. 선택적으로 weight 컬럼 포함. |
| **필수 컬럼** | 날짜(date 계열), 종목 식별자(ticker/종목명 계열), 일별 수익률(return 계열) |
| **선택 컬럼** | 보유 비중(weight 계열) — 있으면 비중 분석 활성화. 자산군(asset class 계열) — 있으면 자산군별 비중 추가 표시. |
| **사전 조건** | 사용자가 Skillfolio 웹 서비스에 접속한 상태. 파일 크기 10MB 이하. |
| **주요 흐름** | 1. 사용자가 CSV 파일을 드래그 앤 드롭으로 업로드한다. 2. 시스템이 컬럼 목록과 상위 5행 미리보기를 표시한다. 3. AI가 컬럼 의미를 추론하고 신뢰도와 함께 매핑 결과를 제안한다. 4. 고신뢰도(≥0.8) 항목은 자동 매핑 확인 화면으로, 중간 신뢰도(0.5~0.8) 항목은 개별 확인 요청으로, 저신뢰도(<0.5) 항목은 수동 선택 UI로 표시한다. 5. 사용자가 매핑을 확정한다. 6. 사용자가 자연어로 "이 포트폴리오가 괜찮은지 분석해줘"라고 입력한다. 7. 시스템이 목적 모호성을 감지하고 UC-02 흐름으로 분기한다. 8. 사용자가 "자동 진단"을 선택한다. 9. 시스템이 데이터 충분성을 검사하고 가능/제한적 가능/불가 분류 결과를 표시한다. 10. 사용자가 확인 후 대시보드 생성을 진행한다. 11. 시스템이 Skills.md 규칙에 따라 분석을 수행하고 30초 이내에 대시보드를 자동 생성한다. 12. AI 애널리스트 패널이 Skills.md 템플릿 기반으로 분석 요약을 표시한다. 13. 사용자가 Skills 로그를 열어 규칙 추적을 확인한다. |
| **대안 흐름** | 매핑 수정이 필요한 경우: 사용자가 AI 제안을 수정하고 확정한다. 수정된 매핑은 Column Mapping Log에 사용자 확인 여부와 함께 기록된다. 분석 목적을 사전 목록에서 선택하는 경우: UC-02 분기 없이 즉시 충분성 검사로 진행된다. |
| **데이터 부족 분기** | 1. 일별 수익률 컬럼이 없고 가격 컬럼이 있는 경우: 가격에서 수익률을 역산하여 성과·드로다운·리스크 분석을 수행한다. 2. 비중 컬럼이 없는 경우: 비중 분석 패널에 "보유 비중 데이터 없음 — weight 또는 holdings 컬럼 포함 파일 추가 시 분석 가능"이라고 표시한다. 비중 분석 없이 나머지 분석은 수행한다. 3. 데이터 기간이 30거래일 미만인 경우: 연환산 변동성은 계산하되 통계적 신뢰성 경고를 포함한다. 60거래일 미만이면 경고를 표시한다. 1년(252거래일) 미만이면 CAGR을 표시하지 않는다. |
| **목적 모호성 분기** | 사용자 입력이 "분석해줘", "괜찮은지 봐줘" 등 포괄적인 경우: AI 애널리스트 패널이 "성과 분석, 리스크 분석, 비중 분석, 자동 진단" 중 선택을 제시한다. 사용자가 선택하지 않으면 분석이 진행되지 않는다. 자동 진단을 선택하면 P0 가능 분석 전체를 수행한다. |
| **가능한 분석** | 성과 분석(누적 수익률, 기간별 수익률, CAGR — 252거래일 이상 시), 드로다운 분석(MDD, 드로다운 시계열 차트, 회복 기간), 리스크 분석(연환산 변동성), 비중 분석(weight 컬럼이 있는 경우) |
| **제한적 가능 분석** | 리스크 분석(연환산 변동성) — 데이터 기간이 30~60거래일 구간이면 통계적 신뢰성 경고 포함. CAGR — 252거래일 미만이면 "주의: 1년 미만 데이터 기준 연환산" 레이블과 함께 선택적 표시. |
| **불가능한 분석** | 샤프 지수(P1 — 무위험 수익률 가정 미포함), 집중도 분석/HHI(P1 — 보유 비중 데이터 기반이며 P0 분석 범위 외), 벤치마크 비교(P1 — 벤치마크 수익률 데이터가 파일에 없음), 트리맵(P1 확장), 거래내역 기반 성과 계산(P1). |
| **필요 추가 데이터** | 샤프 지수: 무위험 수익률 또는 무위험 수익률 컬럼. 집중도 분석: 종목별 보유 비중 컬럼. 벤치마크 비교: 동일 기간 벤치마크 수익률 데이터(사용자 파일 내 포함 또는 별도 파일 업로드 — P1 기능). |
| **대시보드 출력** | 1. 포트폴리오 요약 패널(상단): 분석 기간, 누적 수익률, MDD, 연환산 변동성, 종목 수. 2. 누적 수익률 선 차트: 종목별 및 포트폴리오 전체 누적 수익률 시계열. 3. 드로다운 차트: 포트폴리오 드로다운 시계열 선 차트. MDD 발생 기간 및 회복 기간 표시. 4. 리스크 지표 테이블: 연환산 변동성. 데이터 기간 짧음 경고 포함. 5. 비중 파이차트: weight 컬럼이 있는 경우에만 표시. 없으면 "데이터 부족" 패널. 각 패널 하단에 참조 규칙 ID 표시. |
| **AI 애널리스트 패널 출력** | Skills.md 템플릿 기반으로 다음 섹션을 채운다. 1. 관찰된 수치: "분석 기간 동안 포트폴리오 누적 수익률은 [X]%이며, MDD는 [Y]%입니다. 연환산 변동성은 [Z]%입니다." 2. Skills.md 임계값 기반 위험 경고: MDD가 Skills.md 임계값 이상이면 "MDD가 위험 수준입니다." 3. 데이터 한계 고지: 벤치마크 데이터 없음, 샤프 지수 미계산, 짧은 데이터 기간 경고 등. 4. 불가 분석 안내: "벤치마크 비교는 벤치마크 수익률 데이터가 필요합니다. 파일에 해당 컬럼을 추가하면 분석이 가능합니다." 패널 상단에 "이 해석은 Skills.md 템플릿 기반으로 생성되었으며 비결정적 AI 보조 문장을 포함합니다. 투자 권유가 아닙니다."라는 고지 포함. |
| **Skills 로그 출력** | 1. Column Mapping Log: 원본 컬럼명별 추론 의미·신뢰도·사용자 확인 여부·확정 시각 기록. 2. Sufficiency Check Log: 각 분석 항목별 판정 결과(가능/제한적 가능/불가), 부족 데이터명, 참조 규칙 ID, 제외 사유. 3. Metric Calculation Log: 누적 수익률·MDD·연환산 변동성 각각의 계산 공식·입력값·출력값·참조 규칙 ID. 4. Visualization Rule Log: 선 차트·드로다운 차트·파이차트 선택 사유·참조 규칙 ID. 5. Insight Rule Log: MDD 위험 판정 임계값·위험 판정 결과·AI 보조 문장 사용 여부. |
| **관련 Skills.md 규칙** | 컬럼 매핑 규칙(SKILL-CM), 데이터 충분성 판단 규칙(SKILL-DS), 수익률 계산 규칙(SKILL-PERF), MDD 계산 규칙(SKILL-RISK), 연환산 변동성 계산 규칙(SKILL-RISK), 시각화 선택 규칙(SKILL-VIZ), 인사이트 생성 규칙(SKILL-INS) |
| **수용 기준** | 1. P0 데모 데이터셋(5종목 1년치 일별 수익률 CSV) 업로드 후 30초 이내에 대시보드가 생성된다. 2. 누적 수익률·MDD·연환산 변동성 수치가 동일 입력·매핑·Skills.md 버전에서 재현된다. 3. 각 대시보드 패널 하단에 참조 규칙 ID가 표시된다. 4. Skills 로그 5개 유형이 모두 기록된다. 5. AI 애널리스트 패널에 투자 권유·미래 전망·데이터 외 외부 시장 설명이 포함되지 않는다. 6. 비중 컬럼이 없는 경우 비중 패널에 "데이터 부족" 상태가 명시된다. |
| **실패 / 엣지 케이스** | 파일에 수익률 컬럼이 전혀 없고 가격 컬럼도 없는 경우: "성과·리스크 분석을 위해 일별 수익률 또는 일별 가격 컬럼이 필요합니다."라고 안내. 모든 수익률이 동일한 값(변동성=0)인 경우: 연환산 변동성은 0으로 표시하고 "변동성이 0입니다. 데이터를 확인해주세요." 경고. 수익률이 항상 양수(MDD=0)인 경우: "분석 기간 내 드로다운이 발생하지 않았습니다."라고 표시. 비중 합계가 100%를 벗어나는 경우: 정규화하여 표시하고 "비중 합계를 정규화하여 표시합니다." 경고. |
| **공모전 연관성** | 범용성(25점): 비표준 컬럼명 파일을 AI 매핑으로 처리하는 시연. Skills.md 설계(25점): Skills 로그를 통해 규칙-결과 연결 시연. 대시보드 자동 생성(25점): 업로드에서 대시보드 생성까지 수작업 없이 완주하는 흐름 시연. 바이브코딩 활용(15점): Skills.md 규칙이 대시보드 컴포넌트 생성을 직접 제어한다는 구조 시연. |

### UC-02. 목적이 모호한 경우 — 분석 목적 명확화 흐름

| 항목 | 내용 |
|------|------|
| **우선순위** | P0 |
| **주요 사용자** | 페르소나 A, 페르소나 B |
| **상황** | 사용자가 파일을 업로드하고 매핑을 완료했지만, 분석 목적으로 "이 포트폴리오 한번 봐줘", "뭐가 문제인지 알고 싶어", "괜찮은지 분석해줘"처럼 포괄적인 표현을 입력했다. 시스템이 하나의 특정 분석으로 수렴할 수 없는 상황이다. |

AI 애널리스트 패널이 다음 선택지를 표시한다: "성과 분석(누적 수익률, 기간 수익률)", "리스크 분석(MDD, 연환산 변동성)", "비중 분석(종목별 비중 분포)", "자동 진단(P0 가능 분석 전체 수행)". 4. 사용자가 하나 또는 복수를 선택하거나 "자동 진단"을 선택한다. 5. 선택이 확정되면 데이터 충분성 검사(UC-03 로직)로 진입한다. 6. 이후 UC-01 정상 흐름으로 진행한다. |

| **데이터 부족 분기** | 이 Use Case는 목적 명확화 단계이므로 데이터 충분성 판단은 목적 확정 이후에 수행된다. 단, 업로드된 파일이 분석 가능한 컬럼을 전혀 포함하지 않는 경우에는 목적 선택지 표시 전에 "현재 파일로 수행 가능한 분석이 없습니다."라고 안내한다. |

| **가능한 분석** | 목적이 확정된 이후 데이터 충분성에 따라 결정된다. 이 단계에서는 분석이 수행되지 않는다. |

### UC-03. 데이터 부족 — 가능한 분석과 불가능한 분석 분리

| 항목 | 내용 |
|------|------|
| **우선순위** | P0 |
| **주요 사용자** | 페르소나 A, 페르소나 B |
| **상황** | 사용자가 파일을 업로드하고 분석 목적을 선택했으나, 업로드된 파일이 요청된 분석 항목의 일부 또는 전부를 지원하기에 데이터가 부족한 상황이다. 예를 들어 비중 컬럼이 없거나, 데이터 기간이 매우 짧거나, 수익률 대신 가격 컬럼만 있는 경우이다. |
| **사용자 목적** | 시스템이 오류를 내거나 침묵하지 않고, 어떤 분석이 가능하고 어떤 분석이 불가능한지 명확히 분류하여 안내하며, 가능한 범위에서 부분 대시보드를 생성한다. |
| **입력 데이터** | 컬럼 일부가 누락된 CSV/XLSX. 예: 날짜 + 수익률만 있고 비중·벤치마크·자산군 컬럼이 없는 파일. 또는 데이터 기간이 20거래일처럼 짧은 파일. |
| **필수 컬럼** | 최소한 날짜 컬럼과 수익률 또는 가격 컬럼 중 하나. 이것도 없으면 가능한 분석이 없음을 안내하고 흐름이 종료된다. |
| **선택 컬럼** | 보유 비중(weight 계열) — 있으면 비중 분석 활성화. 벤치마크 수익률 — 있으면 P1 벤치마크 비교 활성화(이 UC에서는 언급만). 자산군 — 있으면 자산군별 비중 표시. |
| **사전 조건** | 파일 업로드 완료. AI 컬럼 매핑 완료. 분석 목적 확정 완료. |
| **주요 흐름** | 1. 분석 목적이 확정된 후 시스템이 데이터 충분성 검사를 수행한다. 2. 각 분석 항목별로 필수 컬럼 존재 여부, 데이터 기간, 결측치 비율을 판단한다. 3. 판정 결과를 "가능", "제한적 가능(경고 포함)", "불가" 세 구간으로 분류하여 표시한다. 4. 불가 항목에는 부족한 데이터 유형과 예시 컬럼명을 명시한다. 5. 사용자에게 "가능한 분석만으로 대시보드 생성" 또는 "추가 데이터 업로드 후 재검사" 중 선택을 제시한다. 6. 사용자가 "가능한 분석만으로 진행"을 선택하면 가능한 분석만으로 대시보드를 생성한다. 7. 불가 항목은 대시보드에 "데이터 부족 — [필요 데이터]" 상태 패널로 표시된다. |
| **대안 흐름** | 사용자가 "추가 데이터 업로드"를 선택하는 경우: 파일 업로드 화면으로 돌아간다. 추가 파일 업로드는 P1 기능(F-UP-03 다중 파일 업로드)에 해당한다. P0에서는 추가 업로드 선택지를 표시하되, 실제 기능은 P1 구현 여부에 따른다. |
| **데이터 부족 분기** | 이 Use Case 전체가 데이터 부족 처리 흐름이다. 판단 기준은 다음과 같다. (가) 수익률 컬럼 없음 + 가격 컬럼 있음: 가격에서 수익률 역산 가능, 성과·드로다운·리스크 분석 가능. (나) 수익률·가격 모두 없음: 성과·드로다운·리스크 분석 불가. (다) 비중 컬럼 없음: 비중 분석 불가. "보유 비중(weight) 또는 평가금액(market value) 컬럼 필요" 안내. (라) 데이터 기간 30거래일 미만: 연환산 변동성 계산 가능하나 통계적 신뢰성 낮음 경고 표시. (마) 데이터 기간 252거래일 미만: CAGR 미표시. 기간 수익률만 표시. (바) 모든 필수 컬럼 없음: "현재 파일로 수행 가능한 분석이 없습니다. 필요한 데이터를 확인하세요." 안내 후 흐름 종료. |
| **목적 모호성 분기** | 해당 없음. 이 UC는 목적이 이미 확정된 이후의 흐름이다. |
| **가능한 분석** | 날짜 + 수익률(또는 가격) 컬럼이 있는 경우: 성과 분석, 드로다운 분석, 연환산 변동성. weight 컬럼도 있는 경우: 비중 분석 추가. |
| **제한적 가능 분석** | 데이터 기간 30~60거래일 구간: 연환산 변동성 계산 가능, 통계적 신뢰성 경고 포함. 데이터 기간 60~252거래일 구간: CAGR 미표시, 기간 수익률만 표시. 비중 합계가 100%를 벗어나는 경우: 정규화 후 표시, 경고 포함. |
| **불가능한 분석** | 수익률·가격 컬럼이 모두 없는 경우: 성과·드로다운·리스크 분석 전체 불가. 비중 컬럼이 없는 경우: 비중 분석 불가. 샤프 지수(P1 — 이 UC에서는 언급만). 벤치마크 비교(P1 — 벤치마크 데이터 없음). 집중도/HHI(P1 — 이 UC에서는 언급만). |
| **필요 추가 데이터** | 비중 분석: weight 또는 market value 컬럼 포함 파일. 벤치마크 비교(P1): 동일 기간 벤치마크 수익률 컬럼 포함. 샤프 지수(P1): 무위험 수익률 또는 무위험 수익률 컬럼. |
| **대시보드 출력** | 가능한 분석 항목의 대시보드 패널이 생성된다. 불가 항목의 패널 위치에는 "데이터 부족 — [필요 데이터명]이 필요합니다"라는 안내가 표시된다. 빈칸이나 오류 화면으로 두지 않는다. |
| **AI 애널리스트 패널 출력** | 데이터 부족 안내: "다음 분석은 현재 파일의 데이터가 부족하여 수행하지 못했습니다. [분석 항목명] — [부족한 데이터 설명]." 가능한 분석에 대한 Skills.md 템플릿 기반 수치 서술. 투자 권유·미래 전망 포함 금지. 제한적 가능 분석의 통계적 신뢰성 경고 명시. |
| **Skills 로그 출력** | Sufficiency Check Log: 각 분석 항목별 판정 결과, 부족 데이터명, 참조 규칙 ID, 제외 사유(excluded_analysis_reason) 기록. Metric Calculation Log: 수행된 분석 항목의 계산 공식·입력값·출력값 기록. Visualization Rule Log: 가능한 분석 항목의 차트 선택 사유 기록. Insight Rule Log: 위험 판정 임계값·결과 기록. |
| **관련 Skills.md 규칙** | 데이터 충분성 판단 규칙(SKILL-DS), 수익률 역산 규칙(SKILL-DS), 최소 데이터 기간 기준(SKILL-DS), 비중 분석 활성화 조건(SKILL-DS) |
| **수용 기준** | 1. 모든 분석 항목에 대해 가능/제한적 가능/불가 중 하나로 판정 결과가 표시된다. 2. 불가 항목에는 부족한 데이터 유형과 예시 컬럼명이 명시된다. 3. 불가 항목에 대해 사용자가 강제 분석을 진행할 수 없다(비활성화). 4. 가능한 분석만으로 부분 대시보드가 생성된다. 5. 불가 항목 위치에 "데이터 부족" 상태 패널이 표시된다(빈칸 없음). 6. 침묵하거나 오류 메시지만 출력하는 경우는 허용되지 않는다. |
| **실패 / 엣지 케이스** | 모든 분석이 불가한 경우: 대시보드가 생성되지 않으며, "현재 파일로 수행 가능한 분석이 없습니다."라고 안내하고 필요한 데이터 목록을 표시한다. 결측치 비율이 100%인 컬럼이 있는 경우: "컬럼 [컬럼명]에 데이터가 없습니다. 이 컬럼을 기반으로 한 분석은 수행할 수 없습니다." 표시. |
| **공모전 연관성** | Skills.md 설계(25점): 데이터 충분성 판단 규칙(SKILL-DS)이 어떤 분석이 실행되고 실행되지 않는지를 제어한다는 것을 Sufficiency Check Log로 증명. 실용성·창의성(10점): 침묵 대신 부분 분석을 제공하는 설계가 서비스 성숙도를 보여준다. 범용성(25점): 데이터 구조가 다르거나 불완전해도 가능한 범위에서 분석을 수행하는 능력. |

| 항목 | 내용 |
|------|------|
| **우선순위** | P0 |
| **주요 사용자** | 페르소나 A (투자 운용 실무자) |
| **상황** | UC-01 흐름을 통해 대시보드가 생성된 후, 사용자는 각 지표가 어떤 규칙에 따라 산출되었는지 확인하고 싶다. 분석 결과를 보고서에 첨부하기 위해 분석 근거를 추적해야 한다. |

| **선택 컬럼** | 없음. |

Skills 로그 패널이 열린다. 5개 유형의 로그 탭이 표시된다. 3. 사용자가 Metric Calculation Log 탭을 열어 MDD 계산의 입력값·출력값·계산 공식·참조 규칙 ID를 확인한다. 4. 사용자가 Visualization Rule Log 탭을 열어 선 차트 선택 사유를 확인한다. 5. 사용자가 Insight Rule Log 탭을 열어 MDD 위험 판정의 임계값 기준을 확인한다. 6. 각 대시보드 패널 하단의 규칙 ID 배지를 클릭하면 해당 패널을 생성한 규칙의 로그 항목이 강조 표시된다. |

| **목적 모호성 분기** | 해당 없음. |
| **가능한 분석** | 로그 열람 대상은 이미 수행된 분석의 기록이다. 추가 분석이 수행되지 않는다. |
| **제한적 가능 분석** | 해당 없음. |
| **불가능한 분석** | 해당 없음. |
| **필요 추가 데이터** | 없음. |
| **대시보드 출력** | 대시보드 패널은 변경되지 않는다. 각 패널 하단의 규칙 ID 배지가 활성화되어 Skills 로그 항목과 연결된다. |

Visualization Rule Log: 선 차트·드로다운 차트·파이차트별 선택 사유(chart_selection_reason), 참조 규칙 ID. Insight Rule Log: 참조 수치 지표, Skills.md 임계값 규칙, 위험 판정 결과, AI 보조 문장 사용 여부. |

| **공모전 연관성** | Skills.md 설계(25점): Skills 로그가 Skills.md 규칙이 실제로 대시보드를 제어했다는 증거물임을 시연. 바이브코딩 활용(15점): Skills.md → 코드 생성 → 대시보드 출력의 연결 고리를 Skills 로그가 추적 가능하게 만든다. 실용성·창의성(10점): 분석 근거를 투명하게 공개하는 설계가 금융 도메인 신뢰성 요건에 대응한다. |

### UC-05. Holdings/weights 기반 포트폴리오 비중 분석

| **우선순위** | P1 |
| **주요 사용자** | 페르소나 A (투자 운용 실무자) |
| **상황** | 사용자가 일별 수익률 데이터 없이 현재 보유 종목과 비중(또는 평가금액)만 담긴 Holdings CSV를 보유하고 있다. 포트폴리오 구성 분포를 빠르게 확인하고 싶다. |

| **필수 컬럼** | 종목명 또는 티커 컬럼, 보유 비중(%) 또는 평가금액 컬럼 중 하나. |

### UC-06. 사용자 파일 내 벤치마크 데이터가 있을 때 비교 분석

### UC-07. 거래내역 파일 인식 및 거래 요약 표시

## 섹션 5. P0 대표 데모 시나리오

이 시나리오는 Skillfolio의 공모전 데모 경로로, 심사위원이 서비스를 평가하는 기준이 되는 단일하고 가장 강력한 흐름이다. 이 흐름은 30초 이내에 완주할 수 있어야 하며, 한 번도 오류가 발생해서는 안 된다.

**입력 조건:**
- 5개 종목, 1년치(약 252거래일) 일별 수익률 CSV
- 컬럼: 날짜(비표준 가능), 종목명(비표준 가능), 일별 수익률(비표준 가능)
- 선택적으로 weight(보유 비중) 컬럼 포함 가능

---

**흐름 서술:**

사용자는 "수익률_최종_최종.csv"처럼 비표준 파일명을 가진 CSV 파일을 Skillfolio 화면에 드래그 앤 드롭으로 업로드한다. 시스템은 파일을 수신하고 컬럼 목록과 상위 5행의 데이터 미리보기를 즉시 표시한다.

AI가 컬럼 의미를 추론한다. "날짜(서울시간)"는 날짜(신뢰도 0.88)로, "수익률_조정"은 일별 수익률(신뢰도 0.72)로, "종목코드"는 종목 식별자(신뢰도 0.91)로 매핑 결과를 제안한다. 고신뢰도 항목은 자동 확인 요청으로, 중간 신뢰도 항목은 개별 확인 요청 팝업으로 표시된다. 사용자가 "수익률_조정 → 일별 수익률" 매핑을 확인하고 전체 매핑을 확정한다.

사용자가 AI 애널리스트 패널 입력창에 "이 포트폴리오가 괜찮은지 분석해줘"라고 타이핑한다. 시스템이 입력을 분석하고 모호한 목적임을 감지한다. AI 애널리스트 패널이 선택지를 표시한다. "성과 분석", "리스크 분석", "비중 분석", "자동 진단(P0 가능 분석 전체 수행)". 사용자가 "자동 진단"을 선택한다.

시스템이 데이터 충분성 검사를 수행한다. 결과가 표시된다. 성과 분석: 가능. 드로다운 분석: 가능. 리스크 분석(연환산 변동성): 가능. 비중 분석: 가능(weight 컬럼이 있는 경우) 또는 "데이터 부족" 안내(없는 경우). 샤프 지수: 불가(P1). 집중도 분석: 불가(P1). 벤치마크 비교: 불가(벤치마크 데이터 없음, P1). 사용자가 결과를 확인하고 대시보드 생성을 진행한다.

시스템이 Skills.md 규칙에 따라 분석을 수행하고 대시보드를 자동 생성한다. 30초 이내에 화면이 완성된다.

대시보드 좌측에 다음 패널이 표시된다.
- 포트폴리오 요약 패널(최상단): 분석 기간, 누적 수익률 수치, MDD 수치, 연환산 변동성 수치, 종목 수.
- 누적 수익률 선 차트: 5개 종목 및 포트폴리오 전체의 누적 수익률 시계열.
- 드로다운 차트: 포트폴리오 드로다운 시계열. MDD 발생 기간 및 회복 기간.
- 리스크 지표 테이블: 연환산 변동성.
- 비중 파이차트: weight 컬럼이 있는 경우 종목별 비중. 없는 경우 "데이터 부족" 패널.

각 패널 하단에 참조 규칙 ID 배지가 표시된다.

AI 애널리스트 패널 우측에 다음 내용이 표시된다. 상단에 "이 해석은 Skills.md 템플릿 기반으로 생성되었으며 비결정적 AI 보조 문장을 포함합니다. 투자 권유가 아닙니다." 고지. 관찰된 수치 섹션: "분석 기간 [시작일~종료일] 동안 포트폴리오 누적 수익률은 [X]%이며, 최대낙폭(MDD)은 [Y]%입니다. 연환산 변동성은 [Z]%입니다." Skills.md 임계값 기반 위험 경고: MDD가 임계값 이상이면 "MDD가 위험 수준입니다(SKILL-RISK-01 기준)." 데이터 한계 고지: "벤치마크 비교, 샤프 지수는 해당 데이터가 파일에 없어 수행하지 못했습니다." 불가 분석 안내.

사용자가 Skills 로그 아이콘을 클릭한다. Skills 로그 패널이 열린다. 5개 탭이 나타난다. Column Mapping Log: 비표준 컬럼명별 추론 결과와 사용자 확인 기록. Sufficiency Check Log: 성과·드로다운·리스크·비중 가능 판정과 샤프·집중도·벤치마크 불가 판정 및 사유. Metric Calculation Log: 누적 수익률 계산 공식(Π(1+r_t) - 1), MDD 계산 공식(peak_to_trough), 연환산 변동성 계산 공식(σ_daily × √252), 참조 규칙 ID. Visualization Rule Log: 누적 수익률 → 선 차트(SKILL-VIZ-01), 드로다운 → 선 차트(SKILL-VIZ-02), 비중 → 파이차트(SKILL-VIZ-03) 선택 사유. Insight Rule Log: MDD 임계값 기준(SKILL-RISK-01), 위험 판정 결과, AI 보조 문장 사용 여부.

이 흐름은 Skillfolio의 핵심 가치 제안인 "비정형 CSV → Skills.md 규칙 → 재현 가능한 대시보드 + Skills 로그 감사"를 완결된 형태로 보여주는 단일 데모 경로다.

## 섹션 6. P1/P2 확장 시나리오 개요

P0 데모 경로는 "5종목 1년치 일별 수익률 CSV → 성과·MDD·변동성·비중 대시보드 + Skills 로그"로 고정된다. 다음 기능들은 P0 데모에 포함되지 않으며, MVP의 견고성이 확인된 이후 확장 단계에서 구현된다.

**UC-05 Holdings/weights 기반 비중 분석 (P1):** 일별 수익률 없이 보유 종목과 비중만으로 포트폴리오 구성을 시각화하는 시나리오다. P0에서 비중 분석은 일별 수익률 파일에 weight 컬럼이 포함된 경우에 한해 수행된다. 별도 Holdings 파일만으로 비중 분석을 수행하는 흐름은 P1로 분류한다. 트리맵과 집중도/HHI도 이 단계에서 추가된다. 트리거 조건: Holdings 파일 단독 업로드 감지 시.

**UC-06 벤치마크 비교 분석 (P1):** 사용자 파일 내에 벤치마크 수익률 컬럼이 포함된 경우에만 활성화된다. 외부 시장 데이터 자동 연결은 이 시나리오에서도 지원하지 않는다. 초과수익·추적오차·정보비율 계산과 누적 수익률 비교 선 차트가 추가된다. 트리거 조건: AI 컬럼 매핑에서 벤치마크 수익률 컬럼이 감지된 경우.

**UC-07 거래내역 파일 인식 및 거래 요약 (P1):** 매매구분 컬럼이 있는 파일을 거래내역 유형으로 인식하고 거래 목록과 기초 요약을 표시한다. 거래내역만으로 수익률을 계산하는 기능은 현금흐름·수수료·배당·미실현손익 처리 복잡도로 인해 P1 이상에서도 별도 설계가 필요하며 이 Use Case에서는 포함되지 않는다. 트리거 조건: 매매구분 컬럼 감지 시.

**UC-08 Markdown 내보내기 (P1), PDF/PNG 내보내기 (P2):** P0에서는 화면 내 Markdown 리포트 열람까지만 제공한다. 파일 다운로드는 P1(Markdown), P2(PDF, PNG)로 분류된다. 트리거 조건: 대시보드 생성 완료 후 내보내기 버튼 활성화.

## 섹션 8. Stage 5 User Flow 인계 항목

Stage 5 User Flow 작성자는 다음 흐름을 Mermaid 플로우차트로 변환해야 한다.

**1. 정상 P0 흐름 (CSV 업로드 → 대시보드 생성):**
파일 업로드 → 컬럼 목록 및 미리보기 표시 → AI 컬럼 의미 추론 → 신뢰도별 매핑 확인 UI 분기(고신뢰/중간/저신뢰) → 매핑 확정 → 분석 목적 선택 → 충분성 검사 → 가능/제한적 가능/불가 분류 표시 → 대시보드 생성 → 포트폴리오 요약 + 누적 수익률 차트 + MDD 차트 + 변동성 테이블 + 비중 파이차트 → AI 애널리스트 패널 표시.

**2. 목적 모호성 명확화 흐름 (UC-02):**
분석 목적 자연어 입력 → 모호성 감지 분기 → 선택지 표시(성과/리스크/비중/자동 진단) → 사용자 선택 → 충분성 검사 진입. 분기 기준: 단일 분석 항목으로 수렴 가능 여부. 범위 밖 요청 분기: "지원하지 않는 분석" 안내 표시.

**3. 데이터 부족 분기 흐름 (UC-03):**
충분성 검사 실행 → 각 분석 항목별 판정 → 가능/제한적 가능/불가 세 구간 분류 표시 → 사용자 선택(가능한 분석만 진행 / 추가 데이터 업로드) → 가능한 분석만으로 부분 대시보드 생성 → 불가 항목 "데이터 부족" 패널 표시. 가격 컬럼 역산 분기: 수익률 없음 + 가격 있음 시 역산 경로 별도 표시.

**4. Skills 로그 열람 흐름 (UC-04):**
대시보드 생성 완료 → Skills 로그 버튼 클릭 → 로그 패널 열림 → 5개 탭 선택(Column Mapping / Sufficiency Check / Metric Calculation / Visualization Rule / Insight Rule) → 각 탭별 기록 열람 → 패널 규칙 ID 배지 클릭 시 해당 로그 항목 강조 표시.

**5. P1 확장 기능 진입 경계:**
P0 완료 화면에서 "추가 분석" 또는 "P1 기능" 진입 경계를 명시적으로 표시한다. Holdings 단독 업로드 감지 시 UC-05 분기. 벤치마크 컬럼 감지 시 UC-06 분기. 매매구분 컬럼 감지 시 UC-07 분기. Markdown 내보내기 버튼(P1) 및 PDF/PNG 버튼(P2) 활성화 조건. P1/P2 기능 진입 경계가 P0 메인 흐름과 명확히 구분되어 표시되어야 한다.
